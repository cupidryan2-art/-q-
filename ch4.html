    Chapter 4

    Functions & Scope 函数与作用域

    In Python, functions are first-class objects (一等公民) — they can be stored in variables, passed as arguments, and returned from other functions. This single fact unlocks an enormous amount of expressive power, especially in data work.

    Defining Functions with def

    Python — basic function
    def greet(name):
    """Return a greeting string."""   # docstring (文档字符串) — optional but good practice
    return f"Hello, {name}!"

result = greet("Alice")
print(result)   # Hello, Alice!
    C++ Comparison

    Python's def is like a C++ function definition, but with no return-type annotation required (though you can add type hints — more below). There's no function prototype / forward declaration — just define the function before you call it.

    The docstring (triple-quoted string immediately after def) is Python's version of a doxygen comment. It's accessible at runtime via help(greet) or greet.__doc__.

    Type hints (类型注解) — optional but recommended

    Python 3.5+ supports optional type hints. They don't affect runtime behaviour — Python stays dynamically typed — but they enable IDE autocomplete and static analysis tools like mypy.

    Python — type-annotated function
    def add(a: int, b: int) -> int:
    return a + b

def greet(name: str, loud: bool = False) -> str:
    msg = f"Hello, {name}!"
    return msg.upper() if loud else msg

    Parameters and Arguments (参数)

    Positional and keyword arguments

    Python
    def describe(name, age, city):
    print(f"{name}, {age}, from {city}")

# Positional (位置参数) — order matters
describe("Alice", 30, "Beijing")

# Keyword (关键字参数) — order doesn't matter
describe(age=30, city="Beijing", name="Alice")

# Mix — positional must come first
describe("Alice", city="Beijing", age=30)

    Default arguments (默认参数)

    Python
    def connect(host, port=5432, timeout=30):
    print(f"Connecting to {host}:{port} (timeout={timeout}s)")

connect("localhost")               # uses defaults
connect("db.prod.com", port=3306)  # override just port
    Classic Pitfall — Mutable Default Arguments
    Never use a mutable object (list, dict) as a default argument. The default is evaluated once at function definition time, not on each call.
    Python — the trap and the fix
    # WRONG — the list is shared across all calls
def add_item(item, lst=[]):
    lst.append(item)
    return lst

add_item(1)   # [1]
add_item(2)   # [1, 2]  ← NOT [2] — same list!

# CORRECT — use None as sentinel, create inside the function
def add_item(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst

    *args — variable positional arguments

    Python
    def total(*args):
    """Accept any number of positional arguments."""
    print(type(args))   # <class 'tuple'>
    return sum(args)

total(1, 2, 3)       # 6
total(10, 20)         # 30
total()               # 0

# Unpack a list into positional args with *
nums = [1, 2, 3, 4]
print(total(*nums))   # same as total(1, 2, 3, 4)

    **kwargs — variable keyword arguments

    Python
    def configure(**kwargs):
    """Accept any number of keyword arguments."""
    print(type(kwargs))   # <class 'dict'>
    for key, value in kwargs.items():
        print(f"  {key} = {value}")

configure(host="localhost", port=5432, debug=True)

# Combining everything — order must be: positional, *args, keyword, **kwargs
def everything(a, b, *args, sep=" ", **kwargs):
    print(a, b, args, sep, kwargs)
    C++ Comparison
    *args is Python's equivalent of C variadic functions (... / va_list) or C++11's variadic templates, but far simpler — you just get a tuple. **kwargs has no direct C++ equivalent; the closest pattern is passing a std::map<std::string, std::any> or using a builder pattern.

    Return Values (返回值)

    Python
    # Implicit return — returns None
def say_hi():
    print("Hi!")

result = say_hi()   # result is None

# Single return
def square(x):
    return x ** 2

# Multiple return values — actually returns a tuple
def divmod_manual(a, b):
    return a // b, a % b   # parentheses optional

quotient, remainder = divmod_manual(17, 5)
# quotient=3, remainder=2

# Early return (guard clause pattern)
def safe_divide(a, b):
    if b == 0:
        return None          # early exit
    return a / b

    Scope: LEGB Rule (作用域)

    When Python looks up a name, it searches four scopes in order: Local → Enclosing → Global → Built-in. This is the LEGB rule (LEGB 规则).

    Python — scope lookup
    x = "global"        # Global scope (全局作用域)

def outer():
    x = "enclosing"  # Enclosing scope (闭包作用域)

    def inner():
        x = "local"  # Local scope (局部作用域)
        print(x)     # 'local'  — found in Local first

    inner()
    print(x)         # 'enclosing'

outer()
print(x)             # 'global'
    Python — global and nonlocal keywords
    count = 0

def increment():
    global count        # declare intent to modify the global
    count += 1

increment()
increment()
print(count)   # 2

# nonlocal — reach into enclosing (not global) scope
def make_counter():
    n = 0
    def tick():
        nonlocal n      # modify n in the enclosing function
        n += 1
        return n
    return tick

counter = make_counter()
counter()   # 1
counter()   # 2
counter()   # 3
    Best Practice
    Avoid global in most code. It makes functions hard to test and reason about. Prefer passing values as arguments and returning results. nonlocal is fine inside closures.
    C++ Comparison
    Python's global scope ≈ C++ file-scope variables. Python's local scope ≈ C++ function-scope. The key difference: in C++ you can shadow an outer variable just by declaring a new one with the same name. In Python, if you assign to a name inside a function, Python treats it as local throughout that function — even before the assignment. Referencing it before the assignment gives UnboundLocalError, not the global value.

    Lambda Functions (匿名函数)

    A lambda is a small anonymous function defined in a single expression. It can take any number of arguments but contains only one expression — its implicit return value.

    Python — lambda syntax
    # lambda arguments: expression
double  = lambda x: x * 2
add     = lambda x, y: x + y
clamp01 = lambda x: max(0.0, min(1.0, x))

double(5)     # 10
add(3, 4)     # 7
clamp01(1.5)  # 1.0

# Primary use case: inline key/sort functions
pairs = [(1, "b"), (3, "a"), (2, "c")]
pairs.sort(key=lambda p: p[0])         # sort by first element
pairs.sort(key=lambda p: p[1])         # sort by second element

# With map() and filter() — functional style
nums = [1, 2, 3, 4, 5]
doubled  = list(map(lambda x: x * 2, nums))       # [2, 4, 6, 8, 10]
evens    = list(filter(lambda x: x % 2 == 0, nums)) # [2, 4]

# Modern Pythonic alternative — comprehensions are usually clearer:
doubled = [x * 2 for x in nums]
evens   = [x for x in nums if x % 2 == 0]
    C++ Comparison
    Python lambda x: x * 2 ≈ C++ [](int x) { return x * 2; }. Python's lambdas are simpler (one expression only) but capture the enclosing scope automatically — no need to specify [&] or [=]. For multi-statement logic, use a named def instead.

    Pure Functions and Side Effects (纯函数)

    A pure function (纯函数) always returns the same output for the same input, and causes no observable side effects (no modifying external state, no I/O, no printing). This concept matters in data work because pure functions are:

    • Predictable — easy to test, no hidden dependencies
    • Composable — you can chain them safely
    • Parallelisable — no shared mutable state
    Python — pure vs impure
    # IMPURE — modifies external state
total = 0
def add_to_total(x):
    global total
    total += x      # side effect: mutates global

# PURE — takes input, returns output, touches nothing else
def running_total(previous, x):
    return previous + x

# IMPURE — mutates its argument
def normalize_inplace(lst):
    max_val = max(lst)
    for i in range(len(lst)):
        lst[i] /= max_val    # modifies the original list

# PURE — returns a new list
def normalize(lst):
    max_val = max(lst)
    return [x / max_val for x in lst]

data = [10, 20, 50, 100]
scaled = normalize(data)   # data is untouched
print(data)    # [10, 20, 50, 100]
print(scaled)  # [0.1, 0.2, 0.5, 1.0]
    Data Science Note
    Pandas and NumPy are built around this pattern. Most DataFrame operations return a new DataFrame rather than mutating the original — df.dropna(), df.sort_values(), df.rename() all return copies by default. Embrace this: it makes your data pipelines reproducible and debuggable.

    Higher-Order Functions (高阶函数)

    Since functions are first-class objects, you can pass them as arguments to other functions. This is a pattern you'll encounter constantly in Python data libraries.

    Python — functions as arguments
    def apply_twice(f, x):
    """Apply function f to x, then apply f to the result."""
    return f(f(x))

apply_twice(lambda x: x * 2, 3)   # 12  →  3→6→12
apply_twice(str.upper, "hello")    # 'HELLO' (idempotent)

# Returning a function — factory pattern (工厂模式)
def make_multiplier(n):
    def multiply(x):
        return x * n      # n is captured from enclosing scope
    return multiply

triple = make_multiplier(3)
triple(7)     # 21

# This is a closure (闭包) — triple "remembers" that n=3
