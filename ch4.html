    第 4 章

    函数与作用域

    Python 里，函数是一等公民（first-class object）——它们可以被存进变量、当作参数传递、从其他函数里返回。正是这一点，解锁了极其强大的表达能力，在数据工作里尤为如此。

    用 def 定义函数

    Python — 基础函数
    def greet(name):
    """返回一个问候字符串。"""   # docstring（文档字符串）——可选但是好习惯
    return f"你好，{name}！"

result = greet("Alice")
print(result)   # 你好，Alice！
    C++ 对比

    Python 的 def 类似 C++ 的函数定义，但不需要返回类型注解（虽然你可以加——见下文）。没有函数原型/前向声明——只要在调用前定义好函数就行。

    docstring（def 后面紧跟的三引号字符串）是 Python 版的 doxygen 注释。可以通过 help(greet) 或 greet.__doc__ 在运行时访问。

    类型注解（type hints）— 可选但推荐

    Python 3.5+ 支持可选的类型注解。它们不影响运行时行为——Python 依然是动态类型的——但能给 IDE 自动补全和 mypy 等静态分析工具提供信息。

    Python — 带类型注解的函数
    def add(a: int, b: int) -> int:
    return a + b

def greet(name: str, loud: bool = False) -> str:
    msg = f"你好，{name}！"
    return msg.upper() if loud else msg

    参数（parameters）

    位置参数与关键字参数

    Python
    def describe(name, age, city):
    print(f"{name}，{age} 岁，来自 {city}")

# 位置参数（positional）——顺序很重要
describe("Alice", 30, "北京")

# 关键字参数（keyword）——顺序无所谓
describe(age=30, city="北京", name="Alice")

# 混合——位置参数必须在前
describe("Alice", city="北京", age=30)

    默认参数（default arguments）

    Python
    def connect(host, port=5432, timeout=30):
    print(f"连接到 {host}:{port}（超时={timeout}s）")

connect("localhost")               # 使用默认值
connect("db.prod.com", port=3306)  # 只覆盖 port
    经典陷阱 — 可变默认参数
    永远不要用可变对象（list、dict）作为默认参数。默认值只在函数定义时求值一次，不是每次调用时求值。
    Python — 陷阱与正确写法
    # 错误——所有调用共享同一个 list
def add_item(item, lst=[]):
    lst.append(item)
    return lst

add_item(1)   # [1]
add_item(2)   # [1, 2]  ← 不是 [2]——是同一个 list！

# 正确——用 None 作哨兵，在函数内部创建
def add_item(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst

    *args — 可变位置参数

    Python
    def total(*args):
    """接受任意数量的位置参数。"""
    print(type(args))   # <class 'tuple'>
    return sum(args)

total(1, 2, 3)       # 6
total(10, 20)         # 30
total()               # 0

# 用 * 把 list 解包为位置参数
nums = [1, 2, 3, 4]
print(total(*nums))   # 等同于 total(1, 2, 3, 4)

    **kwargs — 可变关键字参数

    Python
    def configure(**kwargs):
    """接受任意数量的关键字参数。"""
    print(type(kwargs))   # <class 'dict'>
    for key, value in kwargs.items():
        print(f"  {key} = {value}")

configure(host="localhost", port=5432, debug=True)

# 组合所有形式——顺序必须是：位置参数, *args, 关键字参数, **kwargs
def everything(a, b, *args, sep=" ", **kwargs):
    print(a, b, args, sep, kwargs)
    C++ 对比
    *args 是 Python 版的 C 可变参数（... / va_list）或 C++11 的变参模板，但简单得多——你直接拿到一个 tuple。**kwargs 在 C++ 里没有直接等价物；最接近的模式是传一个 std::map<std::string, std::any> 或用 Builder 模式。

    返回值

    Python
    # 隐式返回——返回 None
def say_hi():
    print("嗨！")

result = say_hi()   # result 是 None

# 单个返回值
def square(x):
    return x ** 2

# 多个返回值——实际上是返回一个 tuple
def divmod_manual(a, b):
    return a // b, a % b   # 括号可选

quotient, remainder = divmod_manual(17, 5)
# quotient=3, remainder=2

# 提前返回（guard clause 模式）
def safe_divide(a, b):
    if b == 0:
        return None          # 提前退出
    return a / b

    作用域：LEGB 规则

    Python 查找一个名字时，按顺序搜索四个作用域：Local（局部）→ Enclosing（闭包）→ Global（全局）→ Built-in（内置）。这就是 LEGB 规则。

    Python — 作用域查找
    x = "全局"        # 全局作用域（Global）

def outer():
    x = "闭包"    # 闭包作用域（Enclosing）

    def inner():
        x = "局部"  # 局部作用域（Local）
        print(x)   # '局部'——先在 Local 找到了

    inner()
    print(x)       # '闭包'

outer()
print(x)           # '全局'
    Python — global 和 nonlocal 关键字
    count = 0

def increment():
    global count        # 声明要修改全局变量
    count += 1

increment()
increment()
print(count)   # 2

# nonlocal——修改外层函数（而非全局）作用域的变量
def make_counter():
    n = 0
    def tick():
        nonlocal n      # 修改外层函数里的 n
        n += 1
        return n
    return tick

counter = make_counter()
counter()   # 1
counter()   # 2
counter()   # 3
    最佳实践
    大多数代码里要避免 global。它让函数难以测试和推理。优先把值作为参数传入、把结果返回出来。在闭包里用 nonlocal 是没问题的。
    C++ 对比
    Python 的全局作用域 ≈ C++ 的文件作用域变量。Python 的局部作用域 ≈ C++ 的函数作用域。关键区别：C++ 里你可以声明一个同名的新变量来遮蔽外层变量。Python 里，如果你在函数内对一个名字赋值，Python 会把它当作整个函数内的局部变量——包括赋值之前的地方。在赋值前引用它会得到 UnboundLocalError，而不是全局变量的值。

    lambda 匿名函数

    lambda 是用单个表达式定义的小型匿名函数。可以接受任意数量的参数，但只能包含一个表达式——这就是它的隐式返回值。

    Python — lambda 语法
    # lambda 参数: 表达式
double  = lambda x: x * 2
add     = lambda x, y: x + y
clamp01 = lambda x: max(0.0, min(1.0, x))

double(5)     # 10
add(3, 4)     # 7
clamp01(1.5)  # 1.0

# 主要用途：内联 key/排序函数
pairs = [(1, "b"), (3, "a"), (2, "c")]
pairs.sort(key=lambda p: p[0])         # 按第一个元素排序
pairs.sort(key=lambda p: p[1])         # 按第二个元素排序

# 和 map()、filter() 配合——函数式风格
nums = [1, 2, 3, 4, 5]
doubled  = list(map(lambda x: x * 2, nums))         # [2, 4, 6, 8, 10]
evens    = list(filter(lambda x: x % 2 == 0, nums)) # [2, 4]

# 现代 Python 风格——推导式通常更清晰：
doubled = [x * 2 for x in nums]
evens   = [x for x in nums if x % 2 == 0]
    C++ 对比
    Python lambda x: x * 2 ≈ C++ [](int x) { return x * 2; }。Python 的 lambda 更简单（只能有一个表达式），但自动捕获外层作用域——不需要指定 [&] 或 [=]。如果需要多条语句，用具名的 def。

    纯函数与副作用

    纯函数（pure function）对同样的输入总是返回同样的输出，且不产生可观察的副作用（不修改外部状态，不做 I/O，不打印）。这个概念在数据工作里很重要，因为纯函数：

    • 可预测——易于测试，没有隐藏依赖
    • 可组合——可以安全地链式调用
    • 可并行——没有共享可变状态
    Python — 纯函数 vs 非纯函数
    # 非纯——修改外部状态
total = 0
def add_to_total(x):
    global total
    total += x      # 副作用：变异全局变量

# 纯——接受输入，返回输出，不碰其他任何东西
def running_total(previous, x):
    return previous + x

# 非纯——变异了参数
def normalize_inplace(lst):
    max_val = max(lst)
    for i in range(len(lst)):
        lst[i] /= max_val    # 修改了原始 list

# 纯——返回新 list
def normalize(lst):
    max_val = max(lst)
    return [x / max_val for x in lst]

data = [10, 20, 50, 100]
scaled = normalize(data)   # data 不受影响
print(data)    # [10, 20, 50, 100]
print(scaled)  # [0.1, 0.2, 0.5, 1.0]
    数据科学注记
    pandas 和 NumPy 都围绕这个模式构建。大多数 DataFrame 操作默认返回新的 DataFrame 而不变异原始对象——df.dropna()、df.sort_values()、df.rename() 都返回副本。拥抱这一点：它让你的数据管道可重现、可调试。

    高阶函数（higher-order functions）

    既然函数是一等公民，你就可以把它们作为参数传给其他函数。这是你在 Python 数据库里会反复遇到的模式。

    Python — 函数作为参数
    def apply_twice(f, x):
    """把函数 f 应用于 x，然后再对结果应用 f。"""
    return f(f(x))

apply_twice(lambda x: x * 2, 3)   # 12  →  3→6→12
apply_twice(str.upper, "hello")    # 'HELLO'（幂等函数）

# 返回函数——工厂模式（factory pattern）
def make_multiplier(n):
    def multiply(x):
        return x * n      # n 从外层作用域捕获
    return multiply

triple = make_multiplier(3)
triple(7)     # 21

# 这是一个闭包（closure）——triple "记住"了 n=3
