    Modules & Ecosystem 模块与生态系统

    Python's real superpower isn't the language — it's the ecosystem. The standard library covers nearly every everyday task, and PyPI (the Python Package Index) hosts over 500,000 third-party packages. This chapter covers how to navigate all of it.

    The import System (导入系统)

    Python organises code into modules (模块) — individual .py files — and packages (包) — directories containing an __init__.py file. You bring them into your namespace with import.

    Python — import forms
    # 1. Import the whole module — access via module.name
import math
math.sqrt(16)     # 4.0
math.pi           # 3.141592653589793

# 2. Import specific names — brings them into local namespace
from math import sqrt, pi
sqrt(16)          # 4.0  (no math. prefix needed)
pi                # 3.141592653589793

# 3. Import with alias (别名) — essential for long names
import numpy as np          # industry-standard alias
import pandas as pd
import matplotlib.pyplot as plt

np.array([1, 2, 3])

# 4. Import everything — avoid in production code
from math import *    # pollutes namespace — hard to trace where names come from

# 5. Conditional import — for optional dependencies
try:
    import ujson as json    # faster JSON library if available
except ImportError:
    import json             # fall back to standard library
    C++ Comparison

    import math ≈ #include <cmath>, but import happens at runtime — the module's code actually executes when you import it for the first time. Subsequent imports of the same module are cached in sys.modules and return instantly, similar to include guards.

    from math import sqrt is roughly equivalent to using std::sqrt; after including the header.

    import numpy as np is like namespace np = numpy; — you're just creating a short alias for the module object.

    How Python finds modules

    Python — the module search path
    import sys

# Python searches these directories in order:
for path in sys.path:
    print(path)

# 1. The directory of the script being run (or '' for REPL/current dir)
# 2. PYTHONPATH environment variable entries
# 3. Standard library directories
# 4. Site-packages (where pip installs third-party packages)

    Standard Library Highlights (标准库)

    Python's standard library is large enough that experienced developers spend years discovering new corners of it. Here are the modules you'll reach for most often in data and scripting work.

    math — Mathematical functions

    Python
    import math

math.sqrt(2)         # 1.4142135623730951
math.floor(3.7)      # 3
math.ceil(3.2)       # 4
math.log(math.e)     # 1.0   — natural log
math.log10(1000)     # 3.0
math.log2(8)         # 3.0
math.sin(math.pi/2)  # 1.0
math.factorial(10)   # 3628800
math.gcd(48, 18)     # 6
math.isfinite(float('inf'))  # False
math.isnan(float('nan'))     # True

    random — Pseudorandom numbers

    Python
    import random

random.random()              # float in [0.0, 1.0)
random.randint(1, 6)         # int in [1, 6] inclusive — roll a die
random.uniform(0.5, 1.5)     # float in [0.5, 1.5]

items = ["apple", "banana", "cherry"]
random.choice(items)         # random element
random.choices(items, k=3)   # 3 random elements WITH replacement
random.sample(items, k=2)    # 2 random elements WITHOUT replacement
random.shuffle(items)        # shuffle in-place

# Reproducible randomness — essential for data science experiments
random.seed(42)
print(random.random())   # always the same value for seed=42

    datetime — Dates and times

    Python
    from datetime import datetime, date, timedelta

now = datetime.now()
print(now)                          # 2024-03-15 14:32:07.123456

today = date.today()
print(today)                        # 2024-03-15

# Parse a date string
d = datetime.strptime("2024-01-15", "%Y-%m-%d")

# Format as string
d.strftime("%B %d, %Y")             # 'January 15, 2024'

# Arithmetic
one_week = timedelta(weeks=1)
next_week = today + one_week

# Difference between dates
delta = datetime(2024, 12, 31) - datetime.now()
print(f"{delta.days} days until New Year")

    os and pathlib — File system

    Python
    import os
from pathlib import Path   # modern, object-oriented approach (推荐)

# pathlib — the modern way (Python 3.4+)
p = Path("data/results.csv")
p.exists()          # True / False
p.name              # 'results.csv'
p.stem              # 'results'
p.suffix            # '.csv'
p.parent            # Path('data')

# Build paths safely (works on Windows and Unix)
data_dir = Path("data")
output   = data_dir / "output" / "report.csv"   # / operator!

# List directory
for f in Path(".").iterdir():
    if f.suffix == ".py":
        print(f)

# Create / delete
Path("logs").mkdir(parents=True, exist_ok=True)

# os module — legacy but still common
os.getcwd()                  # current working directory
os.listdir(".")              # list files
os.path.join("data", "f.csv") # safe path join (use pathlib instead)

    sys — Interpreter internals

    Python
    import sys

sys.version          # '3.12.1 (main, ...)'
sys.platform         # 'darwin' / 'linux' / 'win32'
sys.argv             # command-line arguments: ['script.py', 'arg1', 'arg2']

# Check Python version in code
if sys.version_info < (3, 10):

    pip and Virtual Environments (包管理)

    pip is Python's package installer (包安装工具). It downloads packages from PyPI and installs them. Always use it inside a virtual environment so different projects stay isolated.

    Terminal — common pip commands
    # Install a package
pip install numpy

# Install a specific version
pip install numpy==1.26.4

# Install multiple packages
pip install numpy pandas matplotlib

# Install from a requirements file
pip install -r requirements.txt

# Upgrade a package
pip install --upgrade pandas

# Uninstall
pip uninstall numpy

# List installed packages
pip list

# Show info about a package
pip show pandas

# Freeze current environment to a requirements file
pip freeze > requirements.txt
    Terminal — virtual environment workflow
    # Create
python3 -m venv .venv

# Activate
source .venv/bin/activate    # macOS / Linux
.venv\Scripts\activate       # Windows

# Your prompt changes: (.venv) $

# Install packages — they go into .venv/lib/, not system Python
pip install pandas matplotlib scikit-learn

# Save dependencies
pip freeze > requirements.txt

# Deactivate when done
deactivate

# Recreate the environment on another machine:
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
    Modern Alternative
    uv (from Astral) is a very fast modern replacement for pip + venv, written in Rust. uv init, uv add pandas, uv run script.py. Worth knowing about if you're starting a new project in 2024+.

    The Data Science Stack — A Preview (数据科学生态预览)

    Four libraries form the backbone of Python data science. You'll spend most of Chapters 7–11 with them. Here's what each one is for:

    Terminal — install the stack
    pip install numpy pandas matplotlib scikit-learn

    NumPy (数值计算)

    NumPy provides the ndarray — a fixed-type, C-backed multidimensional array. It's the foundation everything else is built on. The key insight: NumPy operations run in compiled C/Fortran code, so looping over a NumPy array in C is orders of magnitude faster than looping in Python.

    Python — NumPy taste
    import numpy as np

a = np.array([1, 2, 3, 4, 5])
a * 2             # array([2, 4, 6, 8, 10]) — vectorised, no loop!
a ** 2            # array([1, 4, 9, 16, 25])
a.mean()          # 3.0
a.std()           # 1.4142...

# 2D array — matrix operations
m = np.array([[1, 2], [3, 4]])
m.T               # transpose
m @ m             # matrix multiply (Python 3.5+ @ operator)

    pandas (数据分析)

    pandas provides the DataFrame — a table with labelled columns and a row index. Think of it as a spreadsheet you program. If Chapter 3's list-of-dicts was your raw data, a pandas DataFrame is what it becomes after it grows up.

    Python — pandas taste
    import pandas as pd

df = pd.read_csv("sales.csv")   # load a CSV into a DataFrame

df.head()                    # first 5 rows
df.info()                    # column types and null counts
df.describe()                # summary statistics

df["revenue"].mean()         # column mean
df[df["revenue"] > 1000]     # filter rows
df.groupby("region").sum()   # SQL-style GROUP BY

    Matplotlib (数据可视化)

    Matplotlib is Python's foundational plotting library. It can feel verbose, but it's precise. Higher-level libraries like seaborn and pandas' own .plot() wrap it.

    Python — matplotlib taste
    import matplotlib.pyplot as plt

x = [1, 2, 3, 4, 5]
y = [1, 4, 9, 16, 25]

plt.plot(x, y, marker='o')
plt.title("Squares")
plt.xlabel("x")
plt.ylabel("x²")
plt.savefig("squares.png")   # or plt.show() in Jupyter

    scikit-learn (机器学习)

    scikit-learn is the standard ML library for classical algorithms: regression, classification, clustering, dimensionality reduction, model evaluation. It has a beautifully consistent API: every model has .fit(X, y), .predict(X), and .score(X, y).

    Python — scikit-learn taste
    from sklearn.linear_model import LinearRegression
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

model = LinearRegression()
model.fit(X_train, y_train)       # train
predictions = model.predict(X_test)  # infer
score = model.score(X_test, y_test)  # R² score
