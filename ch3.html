    Data Structures 数据结构

    Python ships four built-in collection types: list, tuple, dict, and set. They cover the vast majority of what you'd reach for std::vector, std::unordered_map, or std::set to do in C++. The key question about each one: is it mutable (可变) or immutable (不可变)?

    list (列表) — Ordered, Mutable Sequence

    A list is Python's workhorse container: ordered, zero-indexed, and mutable. It's the closest equivalent to std::vector<T>, except it can hold objects of different types in the same list.

    Python — creating lists
    nums    = [1, 2, 3, 4, 5]
mixed   = [42, "hello", 3.14, True]   # different types in one list
empty   = []
from_range = list(range(5))           # [0, 1, 2, 3, 4]
repeated   = [0] * 5                  # [0, 0, 0, 0, 0]
    Python — indexing and slicing
    fruits = ["apple", "banana", "cherry", "date"]

fruits[0]      # 'apple'   — first element
fruits[-1]     # 'date'    — last element
fruits[1:3]    # ['banana', 'cherry']  — slice [start, stop)
fruits[:2]     # ['apple', 'banana']
fruits[::2]    # ['apple', 'cherry']   — every other element

# Modifying in-place
fruits[0] = "avocado"          # replace
fruits[1:3] = ["blueberry"]    # replace a slice with fewer items
    Python — list methods (CRUD)
    nums = [3, 1, 4, 1, 5, 9]

# Adding
nums.append(2)           # add to end: [3,1,4,1,5,9,2]
nums.insert(0, 99)       # insert at index: [99,3,1,4,1,5,9,2]
nums.extend([6, 5])      # add multiple: appends another list

# Removing
nums.remove(1)           # removes FIRST occurrence of value 1
popped = nums.pop()      # removes and returns last element
popped2 = nums.pop(0)    # removes and returns element at index 0
del nums[2]              # delete by index

# Searching
nums.index(5)            # index of first occurrence (raises ValueError if missing)
nums.count(1)            # count occurrences

# Sorting
nums.sort()              # in-place sort (modifies nums)
nums.sort(reverse=True)  # in-place descending
sorted_copy = sorted(nums)   # returns NEW sorted list, original unchanged
nums.reverse()           # reverse in-place

# Other
len(nums)                # number of elements
nums.copy()              # shallow copy
nums.clear()             # empty the list
    C++ Comparison
    • list.append(x) ≈ vec.push_back(x)
    • list.pop() ≈ vec.pop_back() — but Python's returns the value
    • list.insert(i, x) ≈ vec.insert(vec.begin()+i, x)
    • sorted(lst) ≈ std::sort on a copy — Python's sort is Timsort, stable and O(n log n)
    • Unlike std::vector, Python lists are dynamically typed — no template parameter needed

    Sorting with a key function

    Python
    words = ["banana", "fig", "apple", "elderberry"]

# Sort by length
words.sort(key=len)
# ['fig', 'apple', 'banana', 'elderberry']

# Sort by last character
words.sort(key=lambda w: w[-1])

# Sort a list of dicts by a field
people = [{"name": "Alice", "age": 30}, {"name": "Bob", "age": 25}]
people.sort(key=lambda p: p["age"])
# Bob comes first now

    tuple (元组) — Ordered, Immutable Sequence

    A tuple is like a list but immutable — once created, you can't change its contents. Use tuples when the data shouldn't change: coordinates, RGB colours, function return values, dictionary keys.

    Python — tuples
    point  = (3, 7)
color  = (255, 128, 0)     # RGB — semantically fixed
single = (42,)             # single-element tuple needs trailing comma!
empty  = ()

# Immutable — this raises TypeError:
# point[0] = 99

# Tuple unpacking (元组解包)
x, y = point               # x=3, y=7
r, g, b = color

# Functions returning multiple values — it's just a tuple
def min_max(nums):
    return min(nums), max(nums)   # returns a tuple

lo, hi = min_max([3, 1, 4, 1, 5, 9])
print(lo, hi)   # 1 9

# Swap using tuple unpacking
a, b = 10, 20
a, b = b, a     # no temp variable needed
    C++ Comparison
    Python tuples map well to C++17's std::tuple and structured bindings: auto [lo, hi] = min_max(nums); is the C++ equivalent of lo, hi = min_max(nums). Python tuples can be used as dictionary keys (because they're hashable and immutable); std::vector cannot be a std::unordered_map key for the same reason.

    dict (字典) — Key-Value Map

    Python's dict is a hash map: O(1) average lookup, insertion, and deletion. As of Python 3.7+, dicts preserve insertion order.

    Python — creating and accessing dicts
    person = {"name": "Alice", "age": 30, "city": "Beijing"}
empty  = {}
from_keys = dict(name="Bob", age=25)   # keyword-argument form

# Access
person["name"]          # 'Alice' — raises KeyError if missing
person.get("name")      # 'Alice' — returns None if missing (safe)
person.get("email", "N/A")  # 'N/A' — default value if missing

# Modify
person["age"] = 31      # update existing key
person["email"] = "a@b.com"  # add new key
del person["city"]      # delete key

# Check membership (成员测试)
"name" in person        # True
"salary" in person      # False
    Python — iterating dicts
    person = {"name": "Alice", "age": 30}

# Iterate keys (默认)
for key in person:
    print(key)           # name, age

# Iterate values
for val in person.values():
    print(val)           # Alice, 30

# Iterate key-value pairs
for key, val in person.items():
    print(f"{key}: {val}")

# Useful methods
list(person.keys())      # ['name', 'age']
list(person.values())    # ['Alice', 30]
list(person.items())     # [('name', 'Alice'), ('age', 30)]

# Merge two dicts (Python 3.9+)
defaults = {"color": "red", "size": "M"}
custom   = {"size": "L", "weight": 1.5}
merged   = defaults | custom   # custom values win on conflict
# {'color': 'red', 'size': 'L', 'weight': 1.5}

    Dict comprehension

    Python
    words = ["apple", "fig", "banana"]

# Map each word to its length
lengths = {w: len(w) for w in words}
# {'apple': 5, 'fig': 3, 'banana': 6}

# Invert a dict (swap keys and values)
original = {"a": 1, "b": 2, "c": 3}
inverted = {v: k for k, v in original.items()}
# {1: 'a', 2: 'b', 3: 'c'}
    C++ Comparison
    Python dict ≈ std::unordered_map (hash table, O(1) average). Python dict preserves insertion order; std::unordered_map does not. If you need sorted-key iteration, use sorted(d.keys()) or Python's collections.OrderedDict (mostly redundant now) — or in C++ use std::map.

    set (集合) — Unordered, Unique Values

    A set stores unique values with no guaranteed order. Fast membership testing (O(1)), and supports the full suite of mathematical set operations.

    Python — sets
    primes = {2, 3, 5, 7, 11}
empty  = set()    # NOT {} — that makes an empty dict!

# Deduplication — convert a list with duplicates
nums = [1, 2, 2, 3, 3, 3, 4]
unique = set(nums)    # {1, 2, 3, 4}

# Membership (O(1))
3 in primes           # True
4 in primes           # False

# Modification
primes.add(13)
primes.discard(2)     # remove if present (no error if missing)
primes.remove(3)      # remove — raises KeyError if missing

# Set operations (集合运算)
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

a | b     # union (并集):        {1, 2, 3, 4, 5, 6}
a & b     # intersection (交集): {3, 4}
a - b     # difference (差集):   {1, 2}  — in a but not b
a ^ b     # symmetric diff:      {1, 2, 5, 6}  — in either, not both

a.issubset(b)       # a ⊆ b ?
a.issuperset(b)     # a ⊇ b ?

    Mutability Explained (可变性)

    This distinction is critical, especially once you start passing containers into functions.

    TypeMutable?Hashable?Typical Use
    listYes ✓NoOrdered collection you'll modify
    tupleNoYes ✓Fixed record; dict key; return value
    dictYes ✓NoKey-value lookup
    setYes ✓NoUnique items; membership tests
    frozensetNoYes ✓Immutable set; can be a dict key
    strNoYes ✓Text
    Python — mutation in functions (important!)
    def double_all(lst):
    for i in range(len(lst)):
        lst[i] *= 2   # mutates the original list!

numbers = [1, 2, 3]
double_all(numbers)
print(numbers)   # [2, 4, 6] — the list was changed in place

# To avoid mutation, pass a copy:
double_all(numbers[:])   # or numbers.copy()
    C++ Comparison
    In C++, passing by value copies the container; passing by reference or pointer allows mutation. In Python, all arguments are passed by object reference. Mutable objects (list, dict, set) can be mutated inside functions. Immutable objects (int, str, tuple) cannot. This is similar to C++ const& vs mutable & but implicit — Python gives you no const keyword to enforce immutability at the type level.

    Nested Structures — Foreshadowing DataFrames

    Real data rarely lives in flat arrays. A list of dicts is one of the most common patterns in Python data work — it maps directly to a spreadsheet or database table, and it's exactly the internal representation of a pandas DataFrame.

    Python — list of dicts (table-like data)
    students = [
    {"name": "Alice",   "score": 92, "grade": "A"},
    {"name": "Bob",     "score": 78, "grade": "C"},
    {"name": "Charlie", "score": 85, "grade": "B"},
]

# Access a field
students[0]["name"]    # 'Alice'

# Iterate rows
for s in students:
    print(f"{s['name']}: {s['score']}")

# Filter with comprehension
honor_roll = [s for s in students if s["score"] >= 90]

# Extract a column
names  = [s["name"]  for s in students]
scores = [s["score"] for s in students]

# Sort by score
students.sort(key=lambda s: s["score"], reverse=True)
