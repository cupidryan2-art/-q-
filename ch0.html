    第 0 章

    为什么学 Python？
    C++ 工程师的第一课

    你懂 C++。你知道 vtable 是什么，知道为什么需要 std::move，也知道解引用空指针会发生什么。 Python 在你看来会感觉简单得有点可疑——这不是障眼法，这恰恰就是它的设计目标。

    两种语言，两种哲学

    C++ 诞生于 1980 年代初，起源是"带类的 C"。它的核心原则是：你不为用不到的东西付代价。 这个哲学造就了有史以来最快的通用编程语言之一，也造就了最复杂的语言之一。

    Python 在 1991 年带着截然相反的赌注出现：开发者的时间比 CPU 时间更贵。 它的创造者 Guido van Rossum 想要一门读起来像伪代码、今天就能跑起来的语言——不需要等四十分钟的编译。

    维度C++Python
    执行方式编译成机器码解释执行（CPython 字节码）
    类型系统静态类型，编译期确定动态类型，运行期解析
    内存管理手动（RAII / 智能指针）垃圾回收（引用计数 + GC）
    速度极快较慢——但 NumPy/C 扩展能弥补
    语法详细、显式简洁、可读
    擅长领域系统编程、游戏、实时数据科学、脚本、ML、Web

    解释执行 vs. 编译执行

    运行 C++ 程序时，编译器把源码翻译成直接在 CPU 上跑的机器码，没有中间人。 Python 源码会先被编译成字节码（bytecode）——一种更低级的中间表示——然后由虚拟机（CPython 解释器）逐行执行。

    实际影响是什么？Python 启动快、试错快。你不需要编译，写一行、跑一行、看结果。这让它特别适合数据探索、快速原型和系统间的胶水代码。

    C++ 对比
    Python 解释器最接近的 C++ 类比，是调试器里的"表达式求值"模式——你可以一次执行一条语句并检查状态。Python 的整个运行时就是这样工作的，而且是刻意设计成这样的。

    动态类型 vs. 静态类型

    在 C++ 里，每个变量的类型在编译期就确定了，编译器负责检查。在 Python 里，变量只是一个名字——一个指向对象的标签。对象本身知道自己的类型，变量名不知道。

    C++ — 类型需要显式声明
    int x = 5;
std::string name = "Alice";
double pi = 3.14159;
// x = "hello";  // 编译报错：不能把字符串赋给 int
    Python — 类型在运行时推断
    x = 5
name = "Alice"
pi = 3.14159
x = "hello"    # 完全合法——x 现在指向一个字符串
    注意
    Python 是强类型的——你不能悄悄把字符串和整数相加。只是类型检查发生在运行时而非编译期。1 + "hello" 会抛出 TypeError，不会产生垃圾结果。

    环境配置

    安装 Python

    macOS 和 Linux 通常预装了 python3，也可以通过包管理器安装。Windows 上从 python.org 或 Microsoft Store 安装。 永远用 python3——旧的 python 命令可能指向 Python 2，它已经在 2020 年停止维护了。

    终端 — 验证安装
    python3 --version       # Python 3.12.x
pip3 --version          # pip 24.x

    虚拟环境（virtual environment）

    pip 默认全局安装包——当不同项目需要同一个库的不同版本时，这会造成混乱。虚拟环境是一个隔离目录，有自己的 Python 二进制文件和包。

    终端 — 创建并激活虚拟环境
    python3 -m venv env         # 创建 env/ 目录
source env/bin/activate     # macOS / Linux
# env\Scripts\activate      # Windows（PowerShell）

pip install numpy pandas    # 仅安装在这个环境里
deactivate                  # 退出环境
    C++ 类比
    虚拟环境类似于 C++ 项目里的项目本地 build/ 目录或 conan profile——它隔离依赖，让"项目 A 需要 NumPy 1.x"和"项目 B 需要 NumPy 2.x"互不干扰。

    编辑器选择

    VS Code 配合官方 Python 扩展是最流行的选择——自动补全、集成调试、Jupyter notebook 支持一应俱全。 PyCharm 对大型项目更强大。 做探索性数据工作时，Jupyter notebooks 让你在一个文档里混合代码、输出和 markdown——第 5 章会详细介绍。

    REPL — 你的交互式沙盒

    REPL 是 Read–Eval–Print Loop（读取-求值-打印 循环） 的缩写。你输入一个表达式，Python 立即求值并打印结果，然后等待下一个输入。没有"编译再运行"这个流程。

    终端 — 启动 REPL
    python3
    REPL 会话 — 在 >>> 后输入
    >>> 2 + 2
4
>>> name = "Alice"
>>> f"你好，{name}！"
'你好，Alice！'
>>> type(42)
<class 'int'>
>>> [x ** 2 for x in range(5)]
[0, 1, 4, 9, 16]
>>> exit()    # 或者 Ctrl-D

    REPL 对学习来说无比有用。遇到一个新函数或者不确定某个方法返回什么，直接试就行了。不需要查头文件，不需要编译——立刻得到反馈。

    小技巧
    在 REPL 里，特殊变量 _（下划线）保存上一个表达式的结果。输入 len([1, 2, 3]) 然后再输入 _ * 2——你会得到 6。做快速计算很方便。

    Hello, World — Python 省掉了什么

    让我们把两个版本放在一起看。注意 Python 版本里没有什么东西。

    C++ — hello world
    #include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}
    Python — hello world
    print("Hello, World!")

    Python 相比 C++ 省掉了什么：

    • 没有 #include — 标准库随时可用，需要什么 import 什么
    • 没有 main() — 顶层代码直接执行，脚本从上到下顺序运行
    • 没有类型声明 — 变量不需要提前声明
    • 没有分号 — 换行结束语句（你可以用分号把两条语句放一行，但别这么做）
    • 没有花括号 — 缩进（indentation）就是语法，第 2 章详述
    • 没有 return 0 — 脚本默认正常退出
    Python — 稍微完整一点的第一个脚本
    name = input("你叫什么名字？")   # 从标准输入读取
print(f"你好，{name}！欢迎来到 Python。")
print("2 的 10 次方是：", 2 ** 10)
    输出
    你叫什么名字？Alice
你好，Alice！欢迎来到 Python。
2 的 10 次方是：1024
    C++ 对比
    print() 就是 Python 版的 std::cout，但它自动处理类型转换——不需要 << 运算符重载或 to_string()。Python 的 f"..." 字符串（f-string，格式化字符串字面量）就像 C++20 的 std::format，但 Python 从 3.6 起就是惯用写法了。
