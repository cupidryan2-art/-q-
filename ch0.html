    Chapter 0

    Why Python?
    A C++ Developer's First Look

    You know C++. You know what a vtable is, why std::move exists, and what happens when you dereference a null pointer. Python is going to feel almost suspiciously simple. That's not a trick — it's the design goal.

    Two Languages, Two Philosophies

    C++ was born in the early 1980s as "C with classes." Its guiding principle: you don't pay for what you don't use. That philosophy produced one of the fastest general-purpose languages ever built — and one of the most complex.

    Python arrived in 1991 with the opposite bet: developer time is more expensive than CPU time. Its creator, Guido van Rossum, wanted a language that reads like pseudocode and runs today, not after a forty-minute compile.

    DimensionC++Python
    ExecutionCompiled to machine codeInterpreted (CPython bytecode)
    TypingStatic, declared at compile timeDynamic, resolved at runtime
    MemoryManual (RAII / smart pointers)Garbage-collected (ref counting + GC)
    SpeedExtremely fastSlower — but NumPy/C extensions close the gap
    SyntaxVerbose, explicitMinimal, readable
    StrengthsSystems, games, real-timeData science, scripting, ML, web

    Interpreted vs. Compiled (解释执行 vs. 编译执行)

    When you run a C++ program, the compiler translates your source into machine code that executes directly on the CPU. There's no intermediary. Python source is compiled to bytecode (解释器字节码) — a lower-level representation — and then a virtual machine (the CPython interpreter) runs that bytecode line by line at runtime.

    The practical consequence: Python starts and experiments fast. You don't compile. You write a line, run it, see what happens. This makes it ideal for data exploration, rapid prototyping, and glue code between systems.

    C++ Comparison
    The closest C++ analogy to Python's interpreter is running a debugger in "expression evaluation" mode — you can execute one statement at a time and inspect state. Python's entire runtime works that way, by design.

    Dynamic vs. Static Typing (动态类型 vs. 静态类型)

    In C++, every variable has a type known at compile time. The compiler enforces it. In Python, a variable is just a name — a label pointing at an object. The object knows its own type; the name doesn't.

    C++ — types are declared
    int x = 5;
std::string name = "Alice";
double pi = 3.14159;
// x = "hello";  // compile error: can't assign string to int
    Python — types are inferred at runtime
    x = 5
name = "Alice"
pi = 3.14159
x = "hello"    # perfectly fine — x now points to a string
    Important
    Python is strongly typed (类型安全) — you can't silently add a string to an integer. It's just that types are checked at runtime, not compile time. 1 + "hello" raises a TypeError, it doesn't produce garbage.

    Setting Up Your Environment (环境配置)

    Python itself

    On macOS and Linux, python3 is typically pre-installed or available via your package manager. On Windows, install from python.org or via the Microsoft Store. Always use python3 — the old python command may point to Python 2, which reached end-of-life in 2020.

    Terminal — verify your install
    python3 --version       # Python 3.12.x
pip3 --version          # pip 24.x

    Virtual environments (虚拟环境)

    pip installs packages globally by default — a bad idea when different projects need different versions of the same library. A virtual environment (虚拟环境) is an isolated directory with its own Python binary and packages.

    Terminal — create and activate a venv
    python3 -m venv env         # create an env/ directory
source env/bin/activate     # macOS / Linux
# env\Scripts\activate      # Windows (PowerShell)

pip install numpy pandas    # installs only inside this env
deactivate                  # leave the environment
    C++ Analogy
    A virtual environment is roughly analogous to a project-local build/ directory or a conan profile — it isolates dependencies so that "project A needs NumPy 1.x" and "project B needs NumPy 2.x" don't conflict.

    Editor recommendations

    VS Code with the official Python extension is the most popular choice — excellent autocomplete, integrated debugger, and Jupyter notebook support. JetBrains PyCharm is powerful for larger projects. For exploratory data work, Jupyter notebooks (交互式笔记本) let you mix code, output, and markdown in one document — more on those in Chapter 5.

    The REPL — Your Interactive Sandbox (交互式解释器)

    REPL stands for Read–Eval–Print Loop. You type an expression, Python evaluates it immediately and prints the result, then waits for the next input. There is no "compile and run" cycle.

    Terminal — starting the REPL
    python3
    REPL session — type after >>>
    >>> 2 + 2
4
>>> name = "Alice"
>>> f"Hello, {name}!"
'Hello, Alice!'
>>> type(42)
<class 'int'>
>>> [x ** 2 for x in range(5)]
[0, 1, 4, 9, 16]
>>> exit()    # or Ctrl-D

    The REPL is extraordinarily useful for learning. When you encounter a new function or aren't sure what a method returns, just try it. No header file to check, no compile step — instant feedback.

    Tip
    In the REPL, the special variable _ (underscore) holds the result of the last expression. Type len([1, 2, 3]) and then _ * 2 — you'll get 6. Useful for quick calculations.

    Hello, World — What Python Removes

    Let's look at both side by side. Notice what isn't in the Python version.

    C++ — hello world
    #include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}
    Python — hello world
    print("Hello, World!")

    What Python removes, compared to C++:

    • No #include — the standard library is always available; you import what you need
    • No main() — top-level code runs directly. Scripts just execute top to bottom
    • No type declarations — variables are not declared in advance
    • No semicolons — newlines end statements. (You can use semicolons to put two statements on one line, but don't)
    • No braces for blocks — indentation (缩进) is the syntax. More on this in Chapter 2
    • No return 0 — scripts exit cleanly by default
    Python — a slightly more complete first script
    name = input("What's your name? ")   # read from stdin
print(f"Hello, {name}! Welcome to Python.")
print("2 to the power of 10 is:", 2 ** 10)
    Output
    What's your name? Alice
Hello, Alice! Welcome to Python.
2 to the power of 10 is: 1024
    C++ Comparison
    print() is Python's equivalent of std::cout, but it handles type conversion automatically — you don't need << operator overloads or to_string(). Python's f"..." strings (格式化字符串字面量) are like C++20's std::format, but they've been the idiomatic choice in Python since 3.6.
