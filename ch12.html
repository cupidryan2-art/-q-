    机器学习基础 Machine Learning Fundamentals

    scikit-learn 把机器学习的门槛压得极低：每个模型都只有三个方法——.fit()、.predict()、.score()。这一章教你包裹这三步的整套工作流，以及每一步存在的理由。

    什么是机器学习？

    传统编程：你写规则，喂数据，得答案。
    机器学习：你喂数据和答案，算法自己学规则。

    范式定义典型例子
    监督学习
    Supervised Learning    		 从有标签的输入 X 学习到输出 y 的映射 预测房价、垃圾邮件分类、疾病诊断
    无监督学习
    Unsupervised Learning    		 从无标签数据中发现结构 客户分群（聚类）、异常检测、降维
    回归（Regression） 监督学习：预测一个连续数值 预测薪资、温度、股价
    分类（Classification） 监督学习：预测一个离散类别标签 垃圾/非垃圾、猫/狗、通过/不通过

    机器学习的七步流程

    每个 scikit-learn 项目都遵循相同的七步循环：

    1. 加载 & 探索数据
    2. 清洗缺失值、异常值和错误类型
    3. 特征工程——对类别特征编码、对数值特征缩放、创建衍生列
    4. 切分训练集和测试集
    5. 在训练集上训练模型
    6. 在测试集上评估——训练过程中绝不偷看
    7. 迭代——调超参数、尝试不同模型

    训练集 / 测试集切分

    如果你用训练数据本身来评估模型，你量的是"背题能力"，不是"举一反三的能力"。测试集被锁起来留到最后评估——它模拟的是"从未见过的新数据"。

    Python — train_test_split
    from sklearn.model_selection import train_test_split
import pandas as pd
import numpy as np

df = pd.read_csv("students.csv")

# X = 特征矩阵，y = 目标变量
X = df[["study_hours", "prev_score", "attendance"]]
y = df["final_score"]

# 切分：80% 训练，20% 测试
X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.2,       # 20% 留给测试
    random_state=42,     # 固定随机种子，保证可复现
    stratify=y if y.dtype == bool else None  # 分类任务：保持类别比例
)

print(f"训练集：{len(X_train)} 条 | 测试集：{len(X_test)} 条")

# 经验规则：80/20 或 70/30 切分
# 数据量极少（< 1000 行）时：改用交叉验证
    C++ 类比
    这就是你的模型的单元测试。训练集准确率毫无意义——一个模型完全可以把训练数据背下来，得到 100% 的准确率（这叫过拟合，overfitting）。测试集准确率才是你真正关心的：模型能泛化到它从未见过的数据吗？

    特征缩放 (Feature Scaling)

    很多算法——逻辑回归、SVM、k-NN、神经网络、PCA——对输入特征的量纲非常敏感。如果 salary 的范围是 3 万到 20 万，age 的范围是 22 到 65，salary 会在任何距离或梯度计算中碾压 age。缩放让所有特征站在同一条起跑线上。

    Python — StandardScaler
    from sklearn.preprocessing import StandardScaler, MinMaxScaler

# StandardScaler：z = (x - mean) / std  → 均值=0，标准差=1
scaler = StandardScaler()

# 关键：只在训练集上 fit，然后分别 transform 训练集和测试集
# 如果在全量数据上 fit，就把测试集信息"泄漏"给了模型（数据泄漏）
X_train_scaled = scaler.fit_transform(X_train)   # fit + transform
X_test_scaled  = scaler.transform(X_test)         # 只 transform（用训练集的统计量）

# MinMaxScaler：缩放到 [0, 1]——需要有界输入时用这个
min_max = MinMaxScaler()
X_train_mm = min_max.fit_transform(X_train)

# 决策树和随机森林不需要缩放——它们基于阈值分裂，与尺度无关
# 线性模型、SVM、k-NN、神经网络需要缩放
    数据泄漏——最常见的陷阱 (Data Leakage)
    永远只在训练集上 fit 你的 scaler（以及任何预处理器）。如果你在切分之前对全量数据调用 scaler.fit_transform(X)，测试集的统计信息就污染了 scaler——模型间接"看到"了测试数据。这会虚高评估指标，给你假的安全感。

    线性回归 (Linear Regression)

    Python — LinearRegression
    from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score
import numpy as np

model = LinearRegression()
model.fit(X_train_scaled, y_train)       # 学习系数

# 查看学到的模型参数
print("截距:", model.intercept_)
print("系数:", dict(zip(X_train.columns, model.coef_)))

# 预测
y_pred = model.predict(X_test_scaled)

# 评估
rmse = np.sqrt(mean_squared_error(y_test, y_pred))
mae  = mean_absolute_error(y_test, y_pred)
r2   = r2_score(y_test, y_pred)   # 1.0=完美，0=只会预测均值，<0=比均值还烂

print(f"RMSE: {rmse:.2f}")   # 均方根误差——和目标变量同单位
print(f"MAE:  {mae:.2f}")    # 平均绝对误差——更直观
print(f"R²:   {r2:.3f}")     # 解释方差的比例

    逻辑回归 (Logistic Regression) — 二分类

    别被名字骗了，逻辑回归是个分类算法。它用 sigmoid 函数对样本属于类别 1 的概率建模，然后以 0.5 为阈值做判断。速度快、可解释性强，是任何二分类问题的首选 baseline。

    Python — LogisticRegression
    from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score, classification_report, confusion_matrix

# 二分类目标：通过(1) 或 不通过(0)
y_binary = (df["final_score"] >= 60).astype(int)
X = df[["study_hours", "prev_score", "attendance"]]

X_train, X_test, y_train, y_test = train_test_split(X, y_binary, test_size=0.2, random_state=42)
scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)

model = LogisticRegression(max_iter=1000)
model.fit(X_train_s, y_train)

y_pred  = model.predict(X_test_s)
y_proba = model.predict_proba(X_test_s)[:, 1]  # 类别 1 的概率

print(f"准确率: {accuracy_score(y_test, y_pred):.3f}")

# 混淆矩阵 (Confusion Matrix)
cm = confusion_matrix(y_test, y_pred)
print("混淆矩阵:")
print(cm)
# [[TN  FP]
#  [FN  TP]]

# 完整分类报告
print(classification_report(y_test, y_pred, target_names=["不通过", "通过"]))

    决策树 (Decision Tree)

    决策树通过反复在特征阈值上切分数据，学出一套"if-then"规则。它的最大优势是可解释性——你可以把学到的规则打印出来，一条一条地读。而且决策树不需要特征缩放。

    Python — DecisionTreeClassifier
    from sklearn.tree import DecisionTreeClassifier, export_text

model = DecisionTreeClassifier(
    max_depth=4,          # 限制树的深度——防止过拟合
    min_samples_leaf=5,   # 每个叶节点至少 5 个样本
    random_state=42,
)
model.fit(X_train, y_train)   # 决策树不需要缩放

print(f"训练准确率: {model.score(X_train, y_train):.3f}")
print(f"测试准确率: {model.score(X_test,  y_test):.3f}")
# 如果训练远高于测试，说明过拟合了 → 收紧 max_depth

# 把学到的规则打印成文字
rules = export_text(model, feature_names=list(X.columns))
print(rules)

# 特征重要性——哪些特征最关键？
importances = pd.Series(model.feature_importances_, index=X.columns)
importances.sort_values(ascending=False)

    模型评估指标

    Python — 评估指标
    from sklearn.metrics import (
    accuracy_score, precision_score, recall_score, f1_score,
    confusion_matrix, classification_report, ConfusionMatrixDisplay
)
import matplotlib.pyplot as plt

# 核心分类指标
acc  = accuracy_score(y_test, y_pred)    # (TP+TN) / 总数
prec = precision_score(y_test, y_pred)   # TP / (TP+FP) — 预测为正的里有多少真的是正
rec  = recall_score(y_test, y_pred)      # TP / (TP+FN) — 真正的正例里有多少被找到
f1   = f1_score(y_test, y_pred)          # precision 和 recall 的调和平均

print(f"准确率:  {acc:.3f}")
print(f"精确率:  {prec:.3f}")
print(f"召回率:  {rec:.3f}")
print(f"F1:      {f1:.3f}")

# 什么时候用哪个指标：
# 准确率  — 类别均衡，两种错误代价相同
# 精确率  — 误报（FP）代价高（垃圾邮件过滤：不能误杀正常邮件）
# 召回率  — 漏报（FN）代价高（癌症检测：不能漏掉病例）
# F1      — 类别不均衡，需要在精确率和召回率间取平衡

# 可视化混淆矩阵
fig, ax = plt.subplots(figsize=(4, 4))
ConfusionMatrixDisplay.from_predictions(
    y_test, y_pred,
    display_labels=["不通过", "通过"],
    ax=ax, colorbar=False,
)
plt.tight_layout()
plt.savefig("confusion_matrix.png", dpi=150, bbox_inches="tight")

    交叉验证 (Cross-Validation)

    单次训练/测试切分只给你一个性能估计——可能运气好，也可能运气差，取决于哪些样本恰好落进了测试集。交叉验证（CV）把评估重复 k 次，每次用不同的切分方式，再取平均值，结果要可靠得多。

    Python — cross_val_score
    from sklearn.model_selection import cross_val_score, KFold
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
import numpy as np

# Pipeline（流水线）——把预处理和模型链成一个对象
# 这样可以确保 scaler 只在训练折上 fit，绝不在测试折上 fit
pipe = Pipeline([
    ("scaler", StandardScaler()),
    ("model",  LogisticRegression(max_iter=1000)),
])

# 5 折交叉验证
cv_scores = cross_val_score(pipe, X, y_binary, cv=5, scoring="accuracy")
print(f"CV 得分:     {cv_scores}")
print(f"均值 ± 标准差: {cv_scores.mean():.3f} ± {cv_scores.std():.3f}")
# 例如："均值 ± 标准差: 0.847 ± 0.023"
# 标准差告诉你性能在不同折之间有多稳定

# 分层 K 折——每个折都保持类别比例
from sklearn.model_selection import StratifiedKFold
skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
cv_scores = cross_val_score(pipe, X, y_binary, cv=skf, scoring="f1")
    永远用 Pipeline
    把预处理器和模型包进 Pipeline 不只是方便——这才是正确的做法。把 Pipeline 传给 cross_val_score 时，scaler 会在每个训练折上独立重新 fit，然后只应用到对应的测试折。这是避免交叉验证中数据泄漏的唯一正确方式。凡是包含预处理步骤的工作流，都要用 Pipeline。
