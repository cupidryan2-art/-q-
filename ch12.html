    Machine Learning Fundamentals 机器学习基础

    scikit-learn makes the mechanics of machine learning almost embarrassingly simple: every model has the same three-method API — .fit(), .predict(), .score(). This chapter teaches the workflow that wraps those three calls, and why each step exists.

    What is Machine Learning? (机器学习)

    Classical programming: you write rules, feed in data, get answers.
    Machine learning: you feed in data and answers, and the algorithm learns the rules.

    ParadigmDefinitionExamples
    Supervised Learning
    监督学习    		 Learn a mapping from inputs X to labelled outputs y Predict house price, classify email as spam, diagnose disease
    Unsupervised Learning
    无监督学习    		 Find structure in unlabelled data Customer segmentation (clustering), anomaly detection, dimensionality reduction
    Regression (回归) Supervised: predict a continuous number Predict salary, temperature, stock price
    Classification (分类) Supervised: predict a discrete class label Spam/not spam, dog/cat, pass/fail

    The ML Workflow (机器学习流程)

    Every scikit-learn project follows the same seven-step loop:

    1. Load & explore your data
    2. Clean missing values, outliers, wrong types
    3. Engineer features — encode categoricals, scale numerics, create derived columns
    4. Split into train and test sets
    5. Train a model on the training set
    6. Evaluate on the test set — never peek during training
    7. Iterate — tune hyperparameters, try different models

    Train / Test Split (训练集 / 测试集)

    If you evaluate a model on the same data it trained on, you're measuring memorisation, not generalisation. The test set is held out and used only for final evaluation — it simulates "new, unseen data."

    Python — train_test_split
    from sklearn.model_selection import train_test_split
import pandas as pd
import numpy as np

df = pd.read_csv("students.csv")

# X = features (特征矩阵), y = target (目标变量)
X = df[["study_hours", "prev_score", "attendance"]]
y = df["final_score"]

# Split — 80% train, 20% test
X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.2,       # 20% held out for testing
    random_state=42,     # seed for reproducibility
    stratify=y if y.dtype == bool else None  # preserve class balance for classification
)

print(f"Train: {len(X_train)} samples | Test: {len(X_test)} samples")

# Rule of thumb: 80/20 or 70/30 split
# For very small datasets (< 1000 rows): use cross-validation instead
    Why This Matters
    This is the equivalent of a unit test for your model. Training accuracy is meaningless — a model can memorise the training data and score 100% (this is overfitting, 过拟合). Test accuracy measures what you actually care about: does the model generalise to data it has never seen?

    Feature Scaling (特征缩放)

    Many algorithms — logistic regression, SVM, k-NN, neural networks, PCA — are sensitive to the scale of input features. If salary ranges from 30,000 to 200,000 and age ranges from 22 to 65, salary will dominate any distance or gradient calculation. Scaling puts all features on the same footing.

    Python — StandardScaler
    from sklearn.preprocessing import StandardScaler, MinMaxScaler

# StandardScaler: z = (x - mean) / std  → mean=0, std=1
scaler = StandardScaler()

# CRITICAL: fit on TRAIN data only, then transform both train and test
# Fitting on test data would leak information ("data leakage" 数据泄漏)
X_train_scaled = scaler.fit_transform(X_train)   # fit + transform
X_test_scaled  = scaler.transform(X_test)         # transform only (use train stats)

# MinMaxScaler: scales to [0, 1]  — useful when you need bounded inputs
min_max = MinMaxScaler()
X_train_mm = min_max.fit_transform(X_train)

# Decision trees and random forests do NOT need scaling — they use thresholds
# Linear models, SVM, k-NN, neural networks DO need scaling
    Data Leakage — The Most Common Mistake (数据泄漏)
    Always fit your scaler (and any other preprocessor) on the training set only. If you call scaler.fit_transform(X) on the full dataset before splitting, the test set statistics influence the scaler — the model has indirectly "seen" the test data. This inflates performance metrics and gives you false confidence.

    Linear Regression (线性回归)

    Python — LinearRegression
    from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score
import numpy as np

model = LinearRegression()
model.fit(X_train_scaled, y_train)       # learn coefficients

# Inspect the learned model
print("Intercept:", model.intercept_)
print("Coefficients:", dict(zip(X_train.columns, model.coef_)))

# Predict
y_pred = model.predict(X_test_scaled)

# Evaluate
rmse = np.sqrt(mean_squared_error(y_test, y_pred))
mae  = mean_absolute_error(y_test, y_pred)
r2   = r2_score(y_test, y_pred)          # 1.0 = perfect, 0 = predicts mean, <0 = worse than mean

print(f"RMSE: {rmse:.2f}")   # root mean squared error — same units as target
print(f"MAE:  {mae:.2f}")    # mean absolute error — more interpretable
print(f"R²:   {r2:.3f}")     # proportion of variance explained

    Logistic Regression (逻辑回归) — Binary Classification

    Despite the name, logistic regression is a classification algorithm. It models the probability that a sample belongs to class 1 using the sigmoid function, then thresholds at 0.5. It's fast, interpretable, and a strong baseline for any binary classification problem.

    Python — LogisticRegression
    from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score, classification_report, confusion_matrix

# Binary target: passed (1) or failed (0)
y_binary = (df["final_score"] >= 60).astype(int)
X = df[["study_hours", "prev_score", "attendance"]]

X_train, X_test, y_train, y_test = train_test_split(X, y_binary, test_size=0.2, random_state=42)
scaler = StandardScaler()
X_train_s = scaler.fit_transform(X_train)
X_test_s  = scaler.transform(X_test)

model = LogisticRegression(max_iter=1000)
model.fit(X_train_s, y_train)

y_pred  = model.predict(X_test_s)
y_proba = model.predict_proba(X_test_s)[:, 1]  # probability of class 1

print(f"Accuracy: {accuracy_score(y_test, y_pred):.3f}")

# Confusion matrix (混淆矩阵)
cm = confusion_matrix(y_test, y_pred)
print("Confusion matrix:")
print(cm)
# [[TN  FP]
#  [FN  TP]]

# Full classification report
print(classification_report(y_test, y_pred, target_names=["Failed", "Passed"]))

    Decision Trees (决策树)

    A decision tree learns a series of if-then rules by recursively splitting the data on feature thresholds. It's highly interpretable — you can visualise the exact rules the model learned. It does not require feature scaling.

    Python — DecisionTreeClassifier
    from sklearn.tree import DecisionTreeClassifier, export_text

model = DecisionTreeClassifier(
    max_depth=4,          # limit tree depth — prevents overfitting
    min_samples_leaf=5,   # each leaf needs at least 5 samples
    random_state=42,
)
model.fit(X_train, y_train)   # no scaling needed for trees

print(f"Train accuracy: {model.score(X_train, y_train):.3f}")
print(f"Test  accuracy: {model.score(X_test,  y_test):.3f}")
# If train >> test, the tree is overfitting → increase max_depth restriction

# Print the learned rules as text
rules = export_text(model, feature_names=list(X.columns))
print(rules)

# Feature importance — which features mattered most?
importances = pd.Series(model.feature_importances_, index=X.columns)
importances.sort_values(ascending=False)

    Model Evaluation Metrics (模型评估)

    Python — evaluation metrics
    from sklearn.metrics import (
    accuracy_score, precision_score, recall_score, f1_score,
    confusion_matrix, classification_report, ConfusionMatrixDisplay
)
import matplotlib.pyplot as plt

# Core classification metrics
acc  = accuracy_score(y_test, y_pred)    # (TP+TN) / total
prec = precision_score(y_test, y_pred)   # TP / (TP+FP) — of predicted positives, how many were right?
rec  = recall_score(y_test, y_pred)      # TP / (TP+FN) — of actual positives, how many did we catch?
f1   = f1_score(y_test, y_pred)          # harmonic mean of precision and recall

print(f"Accuracy:  {acc:.3f}")
print(f"Precision: {prec:.3f}")
print(f"Recall:    {rec:.3f}")
print(f"F1:        {f1:.3f}")

# When to use which metric:
# Accuracy — balanced classes, both error types equally costly
# Precision — false positives are expensive (spam filter: don't block real emails)
# Recall    — false negatives are expensive (cancer detection: don't miss cases)
# F1        — imbalanced classes, need balance of precision and recall

# Visual confusion matrix
fig, ax = plt.subplots(figsize=(4, 4))
ConfusionMatrixDisplay.from_predictions(
    y_test, y_pred,
    display_labels=["Failed", "Passed"],
    ax=ax, colorbar=False,
)
plt.tight_layout()
plt.savefig("confusion_matrix.png", dpi=150, bbox_inches="tight")

    Cross-Validation (交叉验证)

    A single train/test split gives you one estimate of performance — which might be lucky or unlucky depending on which samples ended up in the test set. Cross-validation (CV) repeats the evaluation k times on different splits and averages the results, giving a much more reliable estimate.

    Python — cross_val_score
    from sklearn.model_selection import cross_val_score, KFold
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
import numpy as np

# Pipeline (流水线) — chains preprocessing + model into one object
# This ensures the scaler is fit only on training folds, never test folds
pipe = Pipeline([
    ("scaler", StandardScaler()),
    ("model",  LogisticRegression(max_iter=1000)),
])

# 5-fold cross-validation
cv_scores = cross_val_score(pipe, X, y_binary, cv=5, scoring="accuracy")
print(f"CV scores:  {cv_scores}")
print(f"Mean ± std: {cv_scores.mean():.3f} ± {cv_scores.std():.3f}")
# e.g. "Mean ± std: 0.847 ± 0.023"
# The std tells you how stable the performance is across folds

# Stratified K-Fold — preserves class proportions in each fold
from sklearn.model_selection import StratifiedKFold
skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
cv_scores = cross_val_score(pipe, X, y_binary, cv=skf, scoring="f1")
    Always Use a Pipeline
    Wrapping your preprocessor and model in a Pipeline is not just convenient — it's correct. When you pass a Pipeline to cross_val_score, the scaler is re-fit on each training fold separately, and then applied to the corresponding test fold. This is the only way to avoid data leakage in cross-validation. Always use Pipelines for any workflow that includes preprocessing.
