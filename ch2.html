    Control Flow 控制流

    Python's control flow will feel familiar — if, for, while all exist. The difference is what replaces the braces: indentation. And tucked at the end is one of Python's genuine delights: the list comprehension.

    Indentation is Syntax (缩进即结构)

    In C++, braces {} define blocks. Indentation is optional — just a convention enforced by linters. In Python, indentation is the syntax. There are no braces. The interpreter uses whitespace to determine block boundaries.

    C++ — braces define blocks
    if (x > 0) {
    std::cout << "positive" << std::endl;
    std::cout << "still in the block" << std::endl;
}
std::cout << "outside the block" << std::endl;
    Python — indentation defines blocks
    if x > 0:
    print("positive")
    print("still in the block")   # same indent = same block
print("outside the block")        # back to column 0 = exits block
    The Rule
    Use 4 spaces per indent level. Never mix tabs and spaces — Python 3 raises a TabError. Configure your editor to insert spaces when you press Tab. Every modern Python editor does this by default.

    if / elif / else

    Notice the colon : at the end of each condition. This is not optional — it's part of the syntax that signals the start of an indented block.

    Python
    score = 85

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "F"

print(f"Grade: {grade}")   # Grade: B

    Truthiness (真值)

    Python evaluates any value in a boolean context. You don't need == True or != None for most checks.

    Python — falsy values
    # These are all falsy (假值) — evaluate to False in boolean context:
False
None
0          # zero integer
0.0        # zero float
""         # empty string
[]         # empty list
{}         # empty dict
()         # empty tuple
set()      # empty set

# Everything else is truthy (真值)
if []:                # False — empty list
    print("has items")

items = [1, 2, 3]
if items:             # True — non-empty list
    print("has items")   # prints this

    Ternary expression (三元表达式)

    Python — inline conditional
    # Python's ternary: value_if_true if condition else value_if_false
x = 10
label = "even" if x % 2 == 0 else "odd"
print(label)  # even

# C++ equivalent: label = (x % 2 == 0) ? "even" : "odd";

    for Loops — Iterate Over Anything

    Python's for loop iterates over any iterable (可迭代对象) — a list, string, range, dictionary, file, or custom object. There is no index-based for (int i=0; i<n; i++) style loop. You use range() instead.

    Python — range-based for
    # range(stop) — 0 to stop-1
for i in range(5):
    print(i)      # 0 1 2 3 4

# range(start, stop) — start to stop-1
for i in range(2, 6):
    print(i)      # 2 3 4 5

# range(start, stop, step)
for i in range(0, 10, 2):
    print(i)      # 0 2 4 6 8

# Counting down
for i in range(5, 0, -1):
    print(i)      # 5 4 3 2 1
    C++ Comparison
    for i in range(n) is the Python equivalent of for (int i = 0; i < n; i++). for item in collection is like C++11's range-based for: for (auto& item : collection). Python's range() is lazy — it doesn't actually create a list of numbers; it generates them on demand. This is similar to a C++ generator or lazy view.

    Iterating over collections

    Python
    fruits = ["apple", "banana", "cherry"]

# Direct iteration — cleanest
for fruit in fruits:
    print(fruit)

# With index — use enumerate() (枚举)
for i, fruit in enumerate(fruits):
    print(f"{i}: {fruit}")    # 0: apple, 1: banana, 2: cherry

# Iterate two lists in parallel — use zip()
prices = [1.2, 0.8, 2.5]
for fruit, price in zip(fruits, prices):
    print(f"{fruit}: ${price}")

# Iterate dict items
ages = {"Alice": 30, "Bob": 25}
for name, age in ages.items():
    print(f"{name} is {age}")

    while Loops

    Python
    count = 0
while count < 5:
    print(count)
    count += 1    # note: Python has no ++ or -- operators!

# Reading until sentinel value
total = 0
while True:
    val = int(input("Enter number (0 to stop): "))
    if val == 0:
        break
    total += val
print(f"Total: {total}")
    No ++ in Python
    Python has no increment (++) or decrement (--) operators. Use x += 1 and x -= 1. This was a deliberate design choice — ++x and x++ being different was considered a source of confusion.

    break, continue, pass

    Python
    # break — exit the loop entirely
for i in range(10):
    if i == 5:
        break
    print(i)          # prints 0 1 2 3 4

# continue — skip to the next iteration
for i in range(10):
    if i % 2 == 0:
        continue      # skip even numbers
    print(i)          # prints 1 3 5 7 9

# pass — do nothing (placeholder for empty blocks)
for i in range(5):
    pass              # syntactically required, does nothing

# Also used for empty class/function bodies during development
def todo_function():
    pass              # implement later

    else on loops — a Python peculiarity

    Python loops have an optional else clause that runs only if the loop completed without hitting a break. This is unusual and not present in C++, but it's genuinely useful for search patterns.

    Python — loop else (循环的 else 子句)
    target = 7
numbers = [1, 3, 5, 9, 11]

for n in numbers:
    if n == target:
        print(f"Found {target}!")
        break
else:
    # Runs only if we never hit 'break'
    print(f"{target} not found")   # prints this

# Equivalent C++ pattern requires a flag variable — Python avoids that

    List Comprehensions (列表推导式)

    List comprehensions are one of Python's most elegant features. They let you construct a list by expressing a transformation and optional filter in a single readable line. If you find yourself writing a for loop that just builds a list, a comprehension is almost always cleaner.

    Python — comprehension vs. loop
    # C++ style — build a vector of squares
# std::vector squares;
# for (int i = 0; i < 10; i++) squares.push_back(i * i);

# Python loop style
squares = []
for i in range(10):
    squares.append(i ** 2)

# Python comprehension style — same result, one line
squares = [i ** 2 for i in range(10)]
# [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]
    Python — comprehensions with conditions
    # Filter: only even squares
even_squares = [x**2 for x in range(10) if x % 2 == 0]
# [0, 4, 16, 36, 64]

# Transform strings
words = ["hello", "world", "python"]
upper = [w.upper() for w in words]
# ['HELLO', 'WORLD', 'PYTHON']

# Nested comprehension — flatten a 2D list
matrix = [[1,2,3],[4,5,6],[7,8,9]]
flat = [n for row in matrix for n in row]
# [1, 2, 3, 4, 5, 6, 7, 8, 9]

    Dict and set comprehensions

    Python — other comprehension forms
    # Dict comprehension (字典推导式)
words = ["cat", "elephant", "dog"]
lengths = {word: len(word) for word in words}
# {'cat': 3, 'elephant': 8, 'dog': 3}

# Set comprehension (集合推导式) — removes duplicates automatically
nums = [1, 2, 2, 3, 3, 3]
unique_squares = {x**2 for x in nums}
# {1, 4, 9}

# Generator expression (生成器表达式) — lazy, no list created
total = sum(x**2 for x in range(1000))   # no [] brackets!
    When to Use Comprehensions
    Use comprehensions when: you're building a new collection by transforming or filtering an existing one, the logic fits comfortably on one line, and the intent is clear. If you need multiple statements per iteration or complex nesting, a regular loop is more readable.

    Nested Loops and Conditionals

    Python — multiplication table
    for i in range(1, 4):
    for j in range(1, 4):
        print(f"{i}×{j}={i*j}", end="  ")
    print()   # newline after each row

# Output:
# 1×1=1  1×2=2  1×3=3
# 2×1=2  2×2=4  2×3=6
# 3×1=3  3×2=6  3×3=9
    Python — nested conditionals (FizzBuzz)
    for n in range(1, 21):
    if n % 15 == 0:
        print("FizzBuzz")
    elif n % 3 == 0:
        print("Fizz")
    elif n % 5 == 0:
        print("Buzz")
    else:
        print(n)
