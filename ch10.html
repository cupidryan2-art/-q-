    Transform & Aggregate 数据变换

    Once your data is clean, you need to reshape it — grouping rows by category, merging tables, pivoting summaries, and engineering new features. This chapter covers the transformation toolkit that bridges raw data and model-ready features.

    Grouping Data with .groupby()

    .groupby() is pandas' implementation of the SQL GROUP BY clause. The pattern is always: split the DataFrame into groups, apply a function to each group, combine the results. This is the Split-Apply-Combine (分割-应用-合并) paradigm.

    Python — groupby basics
    import pandas as pd

df = pd.read_csv("employees.csv")
# columns: name, dept, salary, years_exp, score

# Single-column group + single aggregation
df.groupby("dept")["salary"].mean()
# dept
# Engineering    95000.0
# HR             62000.0
# Sales          71000.0
# Name: salary, dtype: float64

# Multiple aggregations on one column — .agg()
df.groupby("dept")["salary"].agg(["mean", "median", "std", "count"])

# Multiple columns
df.groupby("dept")[["salary", "score"]].mean()

# Group by multiple columns
df.groupby(["dept", "level"])["salary"].mean()

    Named aggregations (Python 3.8+)

    Python — named agg for clean output
    # Named aggregation: output_col_name = pd.NamedAgg(column, aggfunc)
summary = df.groupby("dept").agg(
    avg_salary  = ("salary", "mean"),
    max_salary  = ("salary", "max"),
    headcount   = ("name",   "count"),
    avg_tenure  = ("years_exp", "median"),
)
# Returns a clean DataFrame with intuitive column names

# Custom aggregation functions
def cv(series):
    """Coefficient of variation (变异系数)."""
    return series.std() / series.mean()

df.groupby("dept")["salary"].agg(["mean", cv])

# Transform — returns same-length Series (for adding group stats back)
df["dept_avg_salary"] = df.groupby("dept")["salary"].transform("mean")
df["salary_ratio"] = df["salary"] / df["dept_avg_salary"]
    C++ Analogy
    groupby + agg is roughly equivalent to iterating over a sorted vector, detecting group boundaries, and computing a running aggregate — but pandas does all the bookkeeping, sorting, and result assembly for you. In SQL terms it's SELECT dept, AVG(salary) FROM employees GROUP BY dept. No temporary containers, no manual iteration.

    Pivot Tables (透视表)

    A pivot table (透视表) reshapes data by spreading one column's values across new columns — turning "long" format into "wide" format. Think of it as a two-dimensional groupby.

    Python — pd.pivot_table
    df = pd.read_csv("sales.csv")
# columns: region, product, quarter, revenue

# Mean revenue by region (rows) × product (columns)
pivot = pd.pivot_table(
    df,
    values  = "revenue",      # what to aggregate
    index   = "region",       # rows
    columns = "product",      # new columns
    aggfunc = "mean",         # how to aggregate (default: mean)
    fill_value = 0,           # replace NaN with 0
)

# Multi-level: rows = region × quarter, columns = product
pd.pivot_table(df,
               values="revenue",
               index=["region", "quarter"],
               columns="product",
               aggfunc="sum",
               margins=True)    # add row/column totals (汇总行)

# Reverse operation: pd.melt — wide → long format (用于机器学习)
long = pd.melt(pivot.reset_index(),
               id_vars="region",
               var_name="product",
               value_name="avg_revenue")

    Merging DataFrames (连接)

    pd.merge() joins two DataFrames on a key column — exactly like SQL JOINs. Understanding the four join types is essential for any multi-table data work.

    Python — pd.merge join types
    employees = pd.DataFrame({
    "emp_id": [1, 2, 3, 4],
    "name":   ["Alice", "Bob", "Charlie", "Dave"],
    "dept_id": [10, 20, 10, 30],
})

departments = pd.DataFrame({
    "dept_id": [10, 20, 40],
    "dept_name": ["Engineering", "Sales", "Marketing"],
})

# INNER JOIN — only rows with matching keys in BOTH tables
pd.merge(employees, departments, on="dept_id", how="inner")
# Alice, Bob, Charlie (Dave's dept_id=30 has no match → dropped)

# LEFT JOIN — all rows from left, matched from right, NaN for no match
pd.merge(employees, departments, on="dept_id", how="left")
# All 4 employees; Dave gets dept_name = NaN

# RIGHT JOIN — all rows from right
pd.merge(employees, departments, on="dept_id", how="right")
# Engineering, Sales appear; Marketing included but no employees match

# OUTER JOIN — all rows from both, NaN where no match
pd.merge(employees, departments, on="dept_id", how="outer")

# Merge on columns with different names
pd.merge(employees, departments,
         left_on="dept_id", right_on="id")

# Merge on index
pd.merge(df1, df2, left_index=True, right_index=True)

    Concatenating DataFrames (pd.concat)

    Python

q1 = pd.read_csv("q1_sales.csv")
q2 = pd.read_csv("q2_sales.csv")

    Applying Functions: .apply() and .map()

    Python
    # .map() — element-wise on a Series (fastest for simple transforms)
df["grade"] = df["score"].map({90: "A", 80: "B", 70: "C"})  # dict mapping
df["score_str"] = df["score"].map(str)   # call a function on each element

# .apply() on a Series — like .map() but for functions returning non-scalar
df["name_parts"] = df["full_name"].apply(lambda n: n.split())

# .apply() on a DataFrame — row-wise (axis=1) or column-wise (axis=0)
def categorise_employee(row):
    if row["score"] >= 90 and row["years_exp"] >= 5:
        return "Senior High Performer"
    elif row["score"] >= 75:
        return "Solid Contributor"
    else:
        return "Needs Support"

df["category"] = df.apply(categorise_employee, axis=1)

# Vectorized alternative — usually MUCH faster than apply for math
# Bad:  df["tax"] = df["salary"].apply(lambda x: x * 0.2)
# Good: df["tax"] = df["salary"] * 0.2

# Use .apply() when you need multi-column logic or non-vectorizable operations
    Performance Note
    .apply() with axis=1 (row-by-row) runs a Python loop under the hood — it's slow on large DataFrames. Always prefer vectorized operations (df["col"] * 2, np.where, pd.cut) when possible. Use .apply() only when the logic genuinely requires row context or external function calls.

    Feature Engineering Basics (特征工程)

    Feature engineering means creating new informative columns from existing ones. It's often the single highest-leverage step in an ML project — better features beat better algorithms.

    Python — common feature engineering patterns
    import pandas as pd
import numpy as np

# 1. Ratio features
df["price_per_sqft"] = df["price"] / df["sqft"]
df["score_per_year"]  = df["total_score"] / df["years_exp"].replace(0, np.nan)

# 2. Binning continuous variables (discretisation 离散化)
df["age_group"] = pd.cut(df["age"],
                          bins=[0, 18, 35, 55, 100],
                          labels=["Youth", "Adult", "Middle", "Senior"])

# 3. Date/time features from datetime columns
df["created_at"] = pd.to_datetime(df["created_at"])
df["hour"]        = df["created_at"].dt.hour
df["day_of_week"] = df["created_at"].dt.dayofweek  # 0=Mon, 6=Sun
df["is_weekend"]  = df["day_of_week"].isin([5, 6]).astype(int)
df["days_since"]  = (pd.Timestamp.now() - df["created_at"]).dt.days

# 4. Text-derived features
df["name_length"]    = df["name"].str.len()
df["word_count"]     = df["description"].str.split().str.len()
df["has_email"]      = df["contact"].str.contains("@").astype(int)

# 5. Interaction features (combining two columns)
df["exp_score"]  = df["years_exp"] * df["performance_score"]

# 6. Log transform — stabilise skewed distributions
df["log_salary"] = np.log1p(df["salary"])   # log(1+x), safe for zero

    Encoding Categorical Variables (类别变量编码)

    Most ML algorithms require numerical input. Categorical columns (strings like "Eng", "Sales") need to be encoded as numbers. The two main approaches:

    Python — one-hot encoding with pd.get_dummies
    df = pd.DataFrame({
    "name": ["Alice", "Bob", "Charlie"],
    "dept": ["Eng", "Sales", "Eng"],
    "city": ["NY", "LA", "NY"],
})

# One-hot encoding (独热编码) — creates binary columns for each category
encoded = pd.get_dummies(df, columns=["dept", "city"])
#    name  dept_Eng  dept_Sales  city_LA  city_NY
#   Alice         1           0        0        1
#     Bob         0           1        1        0
# Charlie         1           0        0        1

# drop_first=True — drop one category to avoid multicollinearity
# (necessary for linear models that require independent features)
pd.get_dummies(df, columns=["dept"], drop_first=True)

# Label encoding — ordinal or when cardinality is high
from sklearn.preprocessing import LabelEncoder
le = LabelEncoder()
df["dept_encoded"] = le.fit_transform(df["dept"])
# Eng→0, Sales→1 (alphabetical) — only meaningful for ordinal data!

# Ordinal encoding with defined order
size_order = {"S": 0, "M": 1, "L": 2, "XL": 3}
df["size_num"] = df["size"].map(size_order)
    One-Hot vs Label Encoding
    Use one-hot encoding for nominal categories with no inherent order (city, colour, department). Use label/ordinal encoding only when there is a meaningful order (small < medium < large). For high-cardinality categories (hundreds of unique values), consider target encoding or embeddings — one-hot creates too many columns.
