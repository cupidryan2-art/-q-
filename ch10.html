    数据变换与聚合

    数据清洗完毕后，你需要重塑它——按类别分组、合并表格、透视汇总、构造新特征。这一章覆盖连接原始数据和模型就绪特征之间的变换工具箱。

    用 .groupby() 分组数据

    .groupby() 是 pandas 对 SQL GROUP BY 子句的实现。模式永远是：分割 DataFrame 为各组，应用函数到每组，合并结果。这就是分割-应用-合并（Split-Apply-Combine）范式。

    Python — groupby 基础
    import pandas as pd

df = pd.read_csv("employees.csv")
# 列：name, dept, salary, years_exp, score

# 单列分组 + 单个聚合
df.groupby("dept")["salary"].mean()
# dept
# 工程部    95000.0
# 人事部    62000.0
# 销售部    71000.0

# 对一列做多个聚合——.agg()
df.groupby("dept")["salary"].agg(["mean", "median", "std", "count"])

# 多列聚合
df.groupby("dept")[["salary", "score"]].mean()

# 按多列分组
df.groupby(["dept", "level"])["salary"].mean()

    具名聚合（Python 3.8+）

    Python — 具名聚合输出更整洁
    # 具名聚合：输出列名 = pd.NamedAgg(列名, 聚合函数)
summary = df.groupby("dept").agg(
    avg_salary  = ("salary",    "mean"),
    max_salary  = ("salary",    "max"),
    headcount   = ("name",      "count"),
    avg_tenure  = ("years_exp", "median"),
)
# 返回列名直观的干净 DataFrame

# 自定义聚合函数
def cv(series):
    """变异系数（Coefficient of Variation）。"""
    return series.std() / series.mean()

df.groupby("dept")["salary"].agg(["mean", cv])

# transform——返回等长 Series（用于把组统计加回原 DataFrame）
df["dept_avg_salary"] = df.groupby("dept")["salary"].transform("mean")
df["salary_ratio"] = df["salary"] / df["dept_avg_salary"]
    C++ 类比
    groupby + agg 大致等价于遍历一个已排序的 vector，检测组边界，并计算滚动聚合——但 pandas 帮你处理了所有的记账、排序和结果组装。用 SQL 来说就是 SELECT dept, AVG(salary) FROM employees GROUP BY dept。没有临时容器，没有手动迭代。

    透视表（pivot table）

    透视表通过把一列的值铺展成新列来重塑数据——把"长"格式变成"宽"格式。把它想成二维的 groupby。

    Python — pd.pivot_table
    df = pd.read_csv("sales.csv")
# 列：region, product, quarter, revenue

# 按地区（行）× 产品（列）的平均收入
pivot = pd.pivot_table(
    df,
    values  = "revenue",      # 聚合什么
    index   = "region",       # 行
    columns = "product",      # 新列
    aggfunc = "mean",         # 如何聚合（默认：mean）
    fill_value = 0,           # 用 0 替换 NaN
)

# 多层级：行 = 地区 × 季度，列 = 产品
pd.pivot_table(df,
               values="revenue",
               index=["region", "quarter"],
               columns="product",
               aggfunc="sum",
               margins=True)    # 添加行/列汇总

# 反向操作：pd.melt——宽格式 → 长格式（机器学习常用）
long = pd.melt(pivot.reset_index(),
               id_vars="region",
               var_name="product",
               value_name="avg_revenue")

    合并 DataFrame（pd.merge）

    pd.merge() 在键列上连接两个 DataFrame——和 SQL JOIN 完全一样。理解四种连接类型是多表数据工作的必备基础。

    Python — pd.merge 连接类型
    employees = pd.DataFrame({
    "emp_id": [1, 2, 3, 4],
    "name":   ["Alice", "Bob", "Charlie", "Dave"],
    "dept_id": [10, 20, 10, 30],
})

departments = pd.DataFrame({
    "dept_id": [10, 20, 40],
    "dept_name": ["工程部", "销售部", "市场部"],
})

# INNER JOIN——只保留两表都有匹配键的行
pd.merge(employees, departments, on="dept_id", how="inner")
# Alice, Bob, Charlie（Dave 的 dept_id=30 无匹配 → 被删除）

# LEFT JOIN——保留左表所有行，右表无匹配的填 NaN
pd.merge(employees, departments, on="dept_id", how="left")
# 4 名员工都在；Dave 的 dept_name = NaN

# RIGHT JOIN——保留右表所有行
pd.merge(employees, departments, on="dept_id", how="right")
# 工程部、销售部出现；市场部包含但没有员工匹配

# OUTER JOIN——保留两表所有行，无匹配的填 NaN
pd.merge(employees, departments, on="dept_id", how="outer")

# 合并不同列名的键
pd.merge(employees, departments,
         left_on="dept_id", right_on="id")

# 按索引合并
pd.merge(df1, df2, left_index=True, right_index=True)

q1 = pd.read_csv("q1_sales.csv")

all_quarters = pd.concat([q1, q2, q3], ignore_index=True)

all_quarters = pd.concat(

# 横向堆叠列（axis=1）——在共享索引上并排拼接
combined = pd.concat([df_features, df_labels], axis=1)

    应用函数：.apply() 与 .map()

    Python
    # .map()——对 Series 逐元素操作（简单变换最快）
df["grade"]    = df["score"].map({90: "A", 80: "B", 70: "C"})  # 字典映射
df["score_str"] = df["score"].map(str)   # 对每个元素调用函数

# .apply() 在 Series 上——类似 .map()，但可以返回非标量
df["name_parts"] = df["full_name"].apply(lambda n: n.split())

# .apply() 在 DataFrame 上——逐行（axis=1）或逐列（axis=0）
def categorise_employee(row):
    if row["score"] >= 90 and row["years_exp"] >= 5:
        return "资深高绩效"
    elif row["score"] >= 75:
        return "稳定贡献者"
    else:
        return "需要支持"

df["category"] = df.apply(categorise_employee, axis=1)

# 向量化替代——数学运算通常比 apply 快得多
# 差：df["tax"] = df["salary"].apply(lambda x: x * 0.2)
# 好：df["tax"] = df["salary"] * 0.2

# 需要多列逻辑或无法向量化的操作时才用 .apply()
    性能注记
    .apply() 加 axis=1（逐行）在底层跑的是 Python 循环——在大 DataFrame 上很慢。尽量优先用向量化操作（df["col"] * 2、np.where、pd.cut）。只有当逻辑确实需要行上下文或外部函数调用时，才用 .apply()。

    特征工程基础

    特征工程意味着从已有列创造新的有信息量的列。这往往是 ML 项目里杠杆最大的单一步骤——更好的特征胜过更好的算法。

    Python — 常见特征工程模式
    import pandas as pd
import numpy as np

# 1. 比率特征
df["price_per_sqft"] = df["price"] / df["sqft"]
df["score_per_year"]  = df["total_score"] / df["years_exp"].replace(0, np.nan)

# 2. 对连续变量分箱（离散化）
df["age_group"] = pd.cut(df["age"],
                          bins=[0, 18, 35, 55, 100],
                          labels=["青少年", "成年", "中年", "老年"])

# 3. 从 datetime 列提取时间特征
df["created_at"] = pd.to_datetime(df["created_at"])
df["hour"]        = df["created_at"].dt.hour
df["day_of_week"] = df["created_at"].dt.dayofweek  # 0=周一，6=周日
df["is_weekend"]  = df["day_of_week"].isin([5, 6]).astype(int)
df["days_since"]  = (pd.Timestamp.now() - df["created_at"]).dt.days

# 4. 文本衍生特征
df["name_length"]   = df["name"].str.len()
df["word_count"]    = df["description"].str.split().str.len()
df["has_email"]     = df["contact"].str.contains("@").astype(int)

# 5. 交互特征（组合两列）
df["exp_score"] = df["years_exp"] * df["performance_score"]

# 6. 对数变换——稳定偏斜分布
df["log_salary"] = np.log1p(df["salary"])   # log(1+x)，对零值安全

    类别变量编码

    大多数 ML 算法要求数值输入。分类列（"工程部"、"销售部" 这样的字符串）需要编码成数字。两种主要方式：

    Python — 用 pd.get_dummies 做独热编码
    df = pd.DataFrame({
    "name": ["Alice", "Bob", "Charlie"],
    "dept": ["工程", "销售", "工程"],
    "city": ["北京", "上海", "北京"],
})

# 独热编码（one-hot encoding）——为每个类别创建二进制列
encoded = pd.get_dummies(df, columns=["dept", "city"])
#    name  dept_工程  dept_销售  city_上海  city_北京
#   Alice         1          0          0          1
#     Bob         0          1          1          0
# Charlie         1          0          0          1

# drop_first=True——删除一个类别以避免多重共线性
# （线性模型需要独立特征时必要）
pd.get_dummies(df, columns=["dept"], drop_first=True)

# 标签编码——有序类别或基数高时
from sklearn.preprocessing import LabelEncoder
le = LabelEncoder()
df["dept_encoded"] = le.fit_transform(df["dept"])
# 工程→0, 销售→1（按字母顺序）——只对有序数据有意义！

# 有定义顺序的序数编码
size_order = {"S": 0, "M": 1, "L": 2, "XL": 3}
df["size_num"] = df["size"].map(size_order)
    独热编码 vs 标签编码
    对没有内在顺序的名义类别（城市、颜色、部门）用独热编码。只有存在有意义顺序时（小 < 中 < 大）才用标签/序数编码。对高基数类别（数百个唯一值），考虑目标编码（target encoding）或 embedding——独热编码会产生太多列。
