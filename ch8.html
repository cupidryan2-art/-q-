    Pandas: Working with Real Data 数据处理

    pandas is the central table in every Python data science workflow. Its two core abstractions — Series and DataFrame — let you manipulate labelled, heterogeneous tabular data with an expressive, SQL-like API. This chapter covers everything you need to load, inspect, select, filter, and modify data.

    DataFrame and Series — The Core Abstractions

    A Series (序列) is a one-dimensional labelled array — essentially a NumPy array with an index. A DataFrame (数据框) is a two-dimensional table: a collection of Series that share the same index. Think of it as a spreadsheet you program.

    Python — Series
    import pandas as pd
import numpy as np

# Creating a Series
s = pd.Series([88, 75, 92, 60], name="score")
print(s)
# 0    88
# 1    75
# 2    92
# 3    60
# Name: score, dtype: int64

# Custom index
s = pd.Series([88, 75, 92, 60],
              index=["Alice", "Bob", "Charlie", "Dave"],
              name="score")
s["Alice"]   # 88
s[s > 80]    # Alice: 88, Charlie: 92  — boolean indexing works on Series too
    Python — creating a DataFrame
    # From a dict of lists — most common approach
df = pd.DataFrame({
    "name":       ["Alice", "Bob", "Charlie", "Dave"],
    "age":        [30, 25, 35, 28],
    "score":      [88, 75, 92, 60],
    "department": ["Eng", "Sales", "Eng", "HR"],
})

# From a list of dicts (as discussed in Chapter 3)
records = [{"name": "Alice", "score": 88}, {"name": "Bob", "score": 75}]
df = pd.DataFrame(records)

# From a NumPy array
arr = np.random.randint(0, 100, (4, 3))
df = pd.DataFrame(arr, columns=["A", "B", "C"])
    C++ Analogy
    A DataFrame is conceptually similar to a std::vector<struct Row> combined with column-wise random access and a rich query API. Unlike a C++ struct array where every row must have the same schema at compile time, a DataFrame can have mixed types per column and you can add/remove columns at runtime.

    Loading Data (加载数据)

    Python — read_csv, read_json
    import pandas as pd

# CSV — the most common format in data science
df = pd.read_csv("data.csv")

# Common read_csv options
df = pd.read_csv(
    "data.csv",
    sep=";",                  # delimiter (default is ",")
    header=0,                 # row number of header (default 0)
    index_col="id",           # use this column as the row index
    usecols=["name","score"], # only load these columns
    dtype={"score": float},   # specify dtypes to avoid inference
    na_values=["N/A", "-"],   # treat these strings as NaN
    encoding="utf-8",
    nrows=1000,               # only read first 1000 rows (great for large files)
)

# JSON
df = pd.read_json("data.json")
df = pd.read_json("https://api.example.com/data.json")  # URL works too

# Excel
df = pd.read_excel("report.xlsx", sheet_name="Q1 Sales")

# From a dict / list (in-memory, useful for testing)
df = pd.DataFrame({"x": [1,2,3], "y": [4,5,6]})

    Exploring Your Data (探索数据)

    The first thing you do with any new dataset is explore it. These five operations answer "what do I have?" in about 30 seconds.

    Python — exploration workflow
    df = pd.read_csv("students.csv")

# Shape (形状) — always start here
df.shape           # (200, 6)  — 200 rows, 6 columns

# First and last rows
df.head()          # first 5 rows (default)
df.head(10)        # first 10 rows
df.tail(3)         # last 3 rows

# Column names and dtypes
df.columns         # Index(['name','age','score','dept', ...])
df.dtypes          # name: object, age: int64, score: float64, ...
df.info()          # concise summary: dtypes, null counts, memory usage

# Descriptive statistics for numerical columns
df.describe()
# count, mean, std, min, 25%, 50%, 75%, max for each numeric col

# Value counts for categorical columns
df["dept"].value_counts()    # Eng: 80, Sales: 60, HR: 60
df["dept"].unique()          # array(['Eng', 'Sales', 'HR'])
df["dept"].nunique()         # 3

# Missing value summary (crucial before any analysis!)
df.isnull().sum()            # count NaN per column
df.isnull().sum() / len(df)  # proportion missing per column

    Selecting Columns

    Python
    df = pd.read_csv("students.csv")

# Single column → Series
df["score"]          # returns a Series
df.score             # attribute access — only works for valid Python identifiers!

# Multiple columns → DataFrame
df[["name", "score"]]        # double brackets — list of column names

# Add a new column
df["passed"] = df["score"] >= 60

# Rename columns
df.rename(columns={"score": "final_score", "dept": "department"}, inplace=True)

# Drop a column
df = df.drop(columns=["temp_col"])    # returns new DataFrame
# or: df.drop(columns=["temp_col"], inplace=True)  # modifies in-place

    Selecting Rows: .loc[] and .iloc[]

    This is where beginners consistently get confused. pandas has two indexers: .loc (label-based) and .iloc (integer position-based). Knowing which to use and when is one of the most important pandas fundamentals.

    Python — loc (label-based, labels inclusive)
    df = pd.DataFrame({
    "name":  ["Alice", "Bob", "Charlie", "Dave"],
    "score": [88, 75, 92, 60],
    "dept":  ["Eng", "Sales", "Eng", "HR"],
}, index=["a", "b", "c", "d"])   # string row labels

# loc uses the INDEX LABEL
df.loc["a"]            # row with label 'a' → Alice's data
df.loc["a", "score"]   # 88  — specific cell
df.loc["a":"c"]        # rows 'a', 'b', 'c'  — NOTE: end is INCLUSIVE with loc!
df.loc[:, "score"]     # all rows, 'score' column
df.loc[["a", "c"], ["name", "score"]]  # specific rows and columns
    Python — iloc (position-based, exclusive end)
    # iloc uses INTEGER POSITIONS — ignores the index labels
df.iloc[0]             # first row (regardless of index label)
df.iloc[0, 1]          # row 0, column 1 → 88
df.iloc[0:3]           # rows 0,1,2  — end is EXCLUSIVE like Python slices!
df.iloc[:, 1]          # all rows, column 1
df.iloc[[0, 2], [0, 1]] # rows 0 and 2, columns 0 and 1
    The Key Difference
    loc = labels. iloc = positions (integer). When your DataFrame has a default integer index (0, 1, 2, …), loc and iloc behave the same for single rows. They differ when your index has custom labels or when slicing: loc["a":"c"] includes "c", while iloc[0:3] excludes position 3. In practice, use .iloc for positional slicing and .loc for label or conditional selection.

    Filtering Rows with Boolean Indexing (布尔索引)

    This is the pandas feature you'll use most. The pattern: create a boolean Series, use it to filter rows. It builds directly on NumPy's boolean indexing from Chapter 7.

    Python — filtering
    df = pd.read_csv("employees.csv")

# Simple filter
df[df["age"] > 30]
df[df["dept"] == "Engineering"]

# Multiple conditions — ALWAYS use & (and) | (or), NOT Python's 'and'/'or'
# AND: parentheses around each condition are required!
df[(df["age"] > 25) & (df["salary"] > 50000)]

# OR
df[(df["dept"] == "Eng") | (df["dept"] == "Sales")]

# NOT
df[~(df["dept"] == "HR")]           # everyone not in HR

# String filtering — .str accessor
df[df["name"].str.startswith("A")]  # names starting with A
df[df["email"].str.contains("@company.com")]

# .isin() — check membership in a list
df[df["dept"].isin(["Eng", "Product", "Design"])]

# .between() — range filter (inclusive on both ends)
df[df["score"].between(60, 90)]

# .query() — SQL-like string syntax (readable for simple conditions)
df.query("age > 25 and salary > 50000")

    Sorting (排序)

    Python
    # Sort by a single column
df.sort_values("score")                    # ascending (default)
df.sort_values("score", ascending=False)   # descending

# Sort by multiple columns (hierarchical sort)
df.sort_values(["dept", "score"], ascending=[True, False])
# First sort by dept A→Z, then within each dept sort score high→low

# Sort by index
df.sort_index()           # sort rows by their index
df.sort_index(axis=1)     # sort columns alphabetically

# Note: sort_values returns a new DataFrame by default
df = df.sort_values("score", ascending=False).reset_index(drop=True)
# reset_index(drop=True) gives the sorted result clean 0,1,2,... indices

    Adding and Modifying Columns (列操作)

    Python
    # Add a column — simple assignment
df["passed"] = df["score"] >= 60
df["score_pct"] = df["score"] / 100.0
df["full_name"] = df["first_name"] + " " + df["last_name"]

# Conditional column — np.where or pd.cut
import numpy as np
df["grade"] = np.where(df["score"] >= 90, "A",
              np.where(df["score"] >= 80, "B",
              np.where(df["score"] >= 70, "C", "F")))

# Binning — pd.cut for equal-width, pd.qcut for equal-frequency
df["score_bin"] = pd.cut(df["score"],
                          bins=[0, 60, 75, 90, 100],
                          labels=["F", "C", "B", "A"])

# Modify a column in-place
df["score"] = df["score"].round(1)         # round to 1 decimal
df["name"]  = df["name"].str.strip().str.title()  # clean strings

# Remove columns
df = df.drop(columns=["temp", "raw_value"])  # drop by name
    Data Science Habit
    In exploratory analysis, always keep a copy of the raw DataFrame: raw = df.copy() before you start cleaning or modifying. pandas operations that return new DataFrames leave df unchanged (unless you use inplace=True), but assignment like df["col"] = ... mutates df directly.
    Saves your progress locally.
