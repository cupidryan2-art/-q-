    Pandas 数据处理

    pandas 是每个 Python 数据科学工作流里的核心桌子。它的两个核心抽象——Series 和 DataFrame——让你用一套表达力极强的类 SQL API 来操作带标签的异构表格数据。这一章覆盖加载、探索、选取、过滤和修改数据你需要知道的一切。

    DataFrame 与 Series — 核心抽象

    Series 是一维带标签数组——本质上是有索引的 NumPy 数组。DataFrame 是二维表格：共享同一个索引的 Series 集合。把它想成你可以编程的电子表格。

    Python — Series
    import pandas as pd
import numpy as np

# 创建 Series
s = pd.Series([88, 75, 92, 60], name="score")
print(s)
# 0    88
# 1    75
# 2    92
# 3    60
# Name: score, dtype: int64

# 自定义索引
s = pd.Series([88, 75, 92, 60],
              index=["Alice", "Bob", "Charlie", "Dave"],
              name="score")
s["Alice"]   # 88
s[s > 80]    # Alice: 88, Charlie: 92——布尔索引在 Series 上同样有效
    Python — 创建 DataFrame
    # 从 list 字典创建——最常用的方式
df = pd.DataFrame({
    "name":       ["Alice", "Bob", "Charlie", "Dave"],
    "age":        [30, 25, 35, 28],
    "score":      [88, 75, 92, 60],
    "department": ["工程", "销售", "工程", "人事"],
})

# 从字典列表创建（第 3 章介绍过的方式）
records = [{"name": "Alice", "score": 88}, {"name": "Bob", "score": 75}]
df = pd.DataFrame(records)

# 从 NumPy 数组创建
arr = np.random.randint(0, 100, (4, 3))
df = pd.DataFrame(arr, columns=["A", "B", "C"])
    C++ 类比
    DataFrame 在概念上类似于 std::vector<struct Row> 加上按列随机访问和丰富查询 API 的组合。和 C++ struct 数组不同——那里每行在编译期必须有相同的 schema——DataFrame 可以按列有不同类型，而且可以在运行时添加/删除列。

    加载数据

    Python — read_csv、read_json
    import pandas as pd

# CSV——数据科学里最常见的格式
df = pd.read_csv("data.csv")

# 常用 read_csv 参数
df = pd.read_csv(
    "data.csv",
    sep=";",                  # 分隔符（默认是","）

    encoding="utf-8",
    nrows=1000,               # 只读前 1000 行（处理大文件很好用）

# JSON

# Excel

    选取行：.loc[] 与 .iloc[]

    这是初学者最容易搞混的地方。pandas 有两种索引器： .loc（基于标签）和 .iloc（基于整数位置）。 知道什么时候用哪个，是 pandas 最重要的基础之一。

    Python — loc（基于标签，终点包含）
    df = pd.DataFrame({
    "name":  ["Alice", "Bob", "Charlie", "Dave"],
    "score": [88, 75, 92, 60],
    "dept":  ["工程", "销售", "工程", "人事"],
}, index=["a", "b", "c", "d"])   # 字符串行标签

# loc 用的是索引标签
df.loc["a"]            # 标签为 'a' 的行 → Alice 的数据
df.loc["a", "score"]   # 88  — 特定单元格
df.loc["a":"c"]        # 'a', 'b', 'c' 三行——注意：loc 的终点是包含的！
df.loc[:, "score"]     # 所有行的 'score' 列
df.loc[["a", "c"], ["name", "score"]]  # 特定行和列
    Python — iloc（基于位置，终点不含）
    # iloc 用的是整数位置——忽略索引标签
df.iloc[0]             # 第一行（不管索引标签是什么）
df.iloc[0, 1]          # 第 0 行，第 1 列 → 88
df.iloc[0:3]           # 第 0,1,2 行——终点不含，和 Python 切片一样！
df.iloc[:, 1]          # 所有行，第 1 列
df.iloc[[0, 2], [0, 1]] # 第 0 和 2 行，第 0 和 1 列
    关键区别
    loc = 标签。iloc = 整数位置。当 DataFrame 有默认整数索引（0, 1, 2, …）时，两者对单行的行为相同。切片时它们有差异：loc["a":"c"] 包含 "c"，而 iloc[0:3] 不含位置 3。实践中：用 .iloc 做位置切片，用 .loc 做标签或条件选取。

    用布尔索引过滤行

    这是你用得最多的 pandas 特性。模式是：创建一个布尔 Series，用它过滤行。直接建立在第 7 章 NumPy 布尔索引的基础上。

    Python — 过滤
    df = pd.read_csv("employees.csv")

# 简单过滤
df[df["age"] > 30]
df[df["dept"] == "工程"]

# 多条件——必须用 & 和 |，不能用 Python 的 'and'/'or'
# 每个条件必须加括号！
df[(df["age"] > 25) & (df["salary"] > 50000)]

# OR
df[(df["dept"] == "工程") | (df["dept"] == "销售")]

# NOT
df[~(df["dept"] == "人事")]           # 不在人事部门的所有人

# 字符串过滤——.str 访问器
df[df["name"].str.startswith("A")]  # 名字以 A 开头
df[df["email"].str.contains("@company.com")]

# .isin()——检查是否在列表里
df[df["dept"].isin(["工程", "产品", "设计"])]

# .between()——范围过滤（两端包含）
df[df["score"].between(60, 90)]

# .query()——类 SQL 字符串语法（简单条件时更易读）
df.query("age > 25 and salary > 50000")

    排序

    Python
    # 按单列排序
df.sort_values("score")                    # 升序（默认）
df.sort_values("score", ascending=False)   # 降序

# 按多列排序（分层排序）
df.sort_values(["dept", "score"], ascending=[True, False])
# 先按部门 A→Z 排，同一部门内按分数从高到低排

# 按索引排序
df.sort_index()           # 按行索引排序
df.sort_index(axis=1)     # 按列名字母顺序排

# 注意：sort_values 默认返回新 DataFrame
df = df.sort_values("score", ascending=False).reset_index(drop=True)
# reset_index(drop=True) 给排序结果一个干净的 0,1,2,... 索引

    添加与修改列

    Python
    # 添加列——简单赋值
df["passed"] = df["score"] >= 60
df["score_pct"] = df["score"] / 100.0
df["full_name"] = df["first_name"] + " " + df["last_name"]

# 条件列——np.where 或 pd.cut
import numpy as np
df["grade"] = np.where(df["score"] >= 90, "A",
              np.where(df["score"] >= 80, "B",
              np.where(df["score"] >= 70, "C", "F")))

# 分箱——pd.cut 等宽，pd.qcut 等频
df["score_bin"] = pd.cut(df["score"],
                          bins=[0, 60, 75, 90, 100],
                          labels=["F", "C", "B", "A"])

# 就地修改列
df["score"] = df["score"].round(1)         # 保留 1 位小数
df["name"]  = df["name"].str.strip().str.title()  # 清洗字符串

# 删除列
df = df.drop(columns=["temp", "raw_value"])
    数据科学好习惯
    在探索性分析里，开始清洗或修改前始终保留一份原始 DataFrame 副本：raw = df.copy()。返回新 DataFrame 的操作不会修改 df（除非用 inplace=True），但 df["col"] = ... 这样的赋值会直接变异 df。
