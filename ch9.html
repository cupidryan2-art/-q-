    Data Cleaning 数据清洗

    Real-world data is messy. Missing values, wrong types, duplicate rows, inconsistent strings, and impossible outliers are the norm, not the exception. Data cleaning is where you spend the most time in any actual project — and where a systematic approach pays the biggest dividends.

    Why Data Cleaning Matters

    A model trained on dirty data learns the dirt. A statistic computed from a column that mixes integers and "N/A" strings gives nonsense. Before any analysis or modelling, you need to understand and resolve data quality issues. The usual workflow:

    1. Detect the problem (missing? duplicated? wrong type? outlier?)
    2. Understand why it exists (collection error? intentional? system artifact?)
    3. Decide how to handle it (drop, fill, fix, flag)
    4. Apply the fix and verify the result

    Handling Missing Values (缺失值)

    pandas uses NaN (Not a Number, from NumPy) as its sentinel for missing data in numeric columns, and None / pd.NA / pd.NaT for other types. They all propagate through operations — adding 5 to a NaN gives NaN — which is how you want missing data to behave.

    Detecting missing values

    Python
    import pandas as pd
import numpy as np

df = pd.read_csv("survey.csv")

# Per-column null counts
df.isnull().sum()
# name        0
# age         3
# income     12
# city        1
# dtype: int64

# Percentage missing — great for quick triage
(df.isnull().sum() / len(df) * 100).round(1)

# Which rows have ANY missing value?
df[df.isnull().any(axis=1)]

# Which rows are COMPLETELY empty? (rare but possible)
df[df.isnull().all(axis=1)]

# Heatmap of missing data (install missingno for a nice visualisation)
# import missingno as msno; msno.matrix(df)

    Dropping missing values — .dropna()

    Python
    # Drop ANY row that has at least one NaN
df.dropna()

# Drop rows where a specific column is NaN (most common)
df.dropna(subset=["income"])

# Drop rows where ALL columns are NaN
df.dropna(how="all")

# Only drop if more than thresh values are non-null
df.dropna(thresh=4)    # keep rows with at least 4 non-null values

# Drop COLUMNS with too many missing values
df.dropna(axis=1, thresh=int(0.8 * len(df)))  # keep cols ≥80% complete

# Important: dropna() returns a NEW DataFrame; use inplace=True or reassign
df = df.dropna(subset=["income"])

    Filling missing values — .fillna()

    Python
    # Fill with a constant
df["age"].fillna(0)
df["city"].fillna("Unknown")

# Fill with column statistics — common imputation strategies
df["income"].fillna(df["income"].mean())     # mean imputation (均值填充)
df["income"].fillna(df["income"].median())   # median — robust to outliers
df["category"].fillna(df["category"].mode()[0])  # most frequent value

# Fill with group means — better than global mean
group_means = df.groupby("dept")["salary"].transform("mean")
df["salary"] = df["salary"].fillna(group_means)

# Forward fill (前向填充) — propagate last valid value forward
# Useful for time series: if Monday's price is known, use it for Tuesday
df["price"].ffill()   # or df["price"].fillna(method="ffill")

# Backward fill
df["price"].bfill()

# Interpolate — linear interpolation between surrounding values
df["temperature"].interpolate(method="linear")
    Which Strategy to Choose?
    Drop when missing data is rare (<5%) and likely random. Mean/median fill for numerical columns when data is roughly symmetric. Median when there are outliers. Mode for categorical columns. Group-based fill when missingness correlates with another column (e.g., missing salary is different in "Intern" vs "Director"). Forward fill for time series.

    Duplicate Detection (重复值)

    Python
    # How many duplicate rows?
df.duplicated().sum()

# Which rows are duplicates? (all columns must match by default)
df[df.duplicated()]

# Check duplicates on specific columns (e.g., same user_id = duplicate)
df.duplicated(subset=["user_id"])

# keep='first' (default): mark all duplicates except first occurrence
# keep='last':  mark all except last
# keep=False:   mark ALL occurrences (including originals)
df[df.duplicated(keep=False)]   # see all rows that have a twin

# Drop duplicates
df = df.drop_duplicates()
df = df.drop_duplicates(subset=["user_id"], keep="last")

    Type Conversion (类型转换)

    When pandas reads a CSV, it infers types. This goes wrong in predictable ways: numeric columns read as strings when they contain "N/A", dates read as strings, boolean columns read as integers. Type conversion is usually the first cleaning step after loading.

    Python — astype() and pd.to_*
    # Check current types
df.dtypes

# Cast with .astype()
df["score"]  = df["score"].astype(float)
df["user_id"] = df["user_id"].astype(str)
df["active"]  = df["active"].astype(bool)

# pd.to_numeric — robust conversion with error handling
# errors='coerce' turns unparseable values into NaN instead of raising
df["income"] = pd.to_numeric(df["income"], errors="coerce")

# pd.to_datetime — date and time parsing
df["created_at"] = pd.to_datetime(df["created_at"])
df["created_at"] = pd.to_datetime(df["date_str"], format="%Y-%m-%d")

# Once a column is datetime, you unlock the .dt accessor:
df["year"]  = df["created_at"].dt.year
df["month"] = df["created_at"].dt.month
df["day_of_week"] = df["created_at"].dt.day_name()   # 'Monday', etc.

# Categorical type — saves memory + unlocks ordering
df["size"] = pd.Categorical(df["size"],
                             categories=["S","M","L","XL"],
                             ordered=True)

    String Cleaning (字符串清洗)

    Text columns are almost always dirty. The .str accessor (字符串访问器) on a Series lets you apply any string method vectorially across all rows — no loop needed.

    Python — .str accessor
    df["name"] = df["name"].str.strip()          # remove leading/trailing spaces
df["name"] = df["name"].str.title()          # 'alice bob' → 'Alice Bob'
df["email"] = df["email"].str.lower()        # normalise case
df["phone"] = df["phone"].str.replace("-", "", regex=False)  # remove dashes

# Extract parts of strings
df["domain"] = df["email"].str.split("@").str[1]   # get domain from email
df["first"]  = df["full_name"].str.split(" ").str[0]

# Check for patterns
df[df["email"].str.contains("@", na=False)]   # na=False avoids NaN crash
df[df["code"].str.match(r"^\d{4}$")]          # exactly 4 digits

# Padding and zero-fill
df["id_padded"] = df["id"].astype(str).str.zfill(6)  # "42" → "000042"

# Replace with regex
df["text"] = df["text"].str.replace(r"\s+", " ", regex=True)  # collapse whitespace

# String cleaning pipeline — chain it all
df["city"] = (df["city"]
              .str.strip()
              .str.lower()
              .str.replace(r"[^\w\s]", "", regex=True)  # remove punctuation
              .str.title())

    Outlier Detection with IQR (异常值检测)

    The IQR (Interquartile Range, 四分位距) method is a robust, distribution-free way to flag outliers. A value is suspicious if it falls more than 1.5 × IQR below Q1 or above Q3. You can tune the multiplier (some fields use 3.0 for "extreme" outliers).

    Python — IQR outlier detection
    import pandas as pd

def flag_outliers_iqr(series: pd.Series, k: float = 1.5) -> pd.Series:
    """Return a boolean mask: True where the value is an outlier."""
    Q1  = series.quantile(0.25)
    Q3  = series.quantile(0.75)
    IQR = Q3 - Q1
    lower = Q1 - k * IQR
    upper = Q3 + k * IQR
    return (series < lower) | (series > upper)

# Apply to the 'salary' column
mask = flag_outliers_iqr(df["salary"])
print(f"Outliers found: {mask.sum()} ({mask.mean()*100:.1f}%)")
df[mask]    # inspect them before deciding what to do

# Options for handling outliers:

# 1. Remove them
df_clean = df[~mask]

# 2. Winsorise (cap, 截断) — replace with boundary value
lower = df["salary"].quantile(0.25) - 1.5 * (df["salary"].quantile(0.75) - df["salary"].quantile(0.25))
upper = df["salary"].quantile(0.75) + 1.5 * (df["salary"].quantile(0.75) - df["salary"].quantile(0.25))
df["salary_capped"] = df["salary"].clip(lower, upper)

# 3. Log transform — compresses large values
import numpy as np
df["salary_log"] = np.log1p(df["salary"])   # log(1+x) handles zeros

    Renaming Columns and Reindexing

    Python
    # Rename specific columns
df.rename(columns={"old_name": "new_name", "qty": "quantity"}, inplace=True)

# Make all column names lowercase with underscores (common cleaning step)
df.columns = df.columns.str.strip().str.lower().str.replace(" ", "_", regex=False)

# Reorder columns
cols = ["id", "name", "score", "grade", "dept"]
df   = df[cols]   # or df.reindex(columns=cols)

# Reset the row index after filtering (avoids gaps like 0,2,5,7...)
df = df.reset_index(drop=True)   # drop=True discards the old index

# Set a column as the index
df = df.set_index("id")   # rows are now keyed by 'id'
df = df.reset_index()     # move index back to a column
    Cleaning Checklist
    Before modelling, run through this in order: (1) shape and dtypes — do they make sense? (2) missing values — how many and why? (3) duplicates — any identical rows? (4) string columns — consistent casing and no leading spaces? (5) date columns — parsed as datetime? (6) numeric columns — any impossible values (negative age, 200% score)?
