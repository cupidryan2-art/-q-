    数据清洗

    真实世界的数据是一团乱麻。缺失值、错误类型、重复行、格式不一致的字符串、不可能出现的异常值——这些是常态，不是例外。 数据清洗是你在实际项目里花时间最多的地方，也是系统化方法回报最丰厚的地方。

    为什么数据清洗如此重要

    用脏数据训练的模型，学到的就是脏规律。对一列混杂了整数和字符串 "N/A" 的数据求统计，得到的是无意义的结果。在任何分析或建模之前，你需要理解并解决数据质量问题。通常的工作流：

    1. 发现问题（缺失？重复？类型错误？异常值？）
    2. 理解原因（采集错误？故意留空？系统产生的？）
    3. 决定处理方式（删除、填充、修正、标记）
    4. 应用修复并验证结果

    处理缺失值（missing values）

    pandas 用 NaN（Not a Number，来自 NumPy）作为数值列缺失数据的哨兵值，其他类型用 None / pd.NA / pd.NaT。它们在运算中会传播——给 NaN 加 5 得到的还是 NaN——这正是你希望缺失数据表现的方式。

    检测缺失值

    Python
    import pandas as pd
import numpy as np

df = pd.read_csv("survey.csv")

# 每列空值计数
df.isnull().sum()
# name        0
# age         3
# income     12
# city        1

# 缺失百分比——快速分类用
(df.isnull().sum() / len(df) * 100).round(1)

# 有任意缺失值的行
df[df.isnull().any(axis=1)]

# 完全为空的行（罕见但可能出现）
df[df.isnull().all(axis=1)]

    删除缺失值 — .dropna()

    Python
    # 删除有任何 NaN 的行
df.dropna()

# 删除特定列为 NaN 的行（最常用）
df.dropna(subset=["income"])

# 删除所有列都是 NaN 的行
df.dropna(how="all")

# 只有非空值不足 thresh 个时才删除
df.dropna(thresh=4)    # 保留至少有 4 个非空值的行

# 删除缺失太多的列
df.dropna(axis=1, thresh=int(0.8 * len(df)))  # 保留 ≥80% 完整的列

# 重要：dropna() 返回新 DataFrame；用 inplace=True 或重新赋值
df = df.dropna(subset=["income"])

    填充缺失值 — .fillna()

    Python
    # 用常数填充
df["age"].fillna(0)
df["city"].fillna("未知")

# 用列统计量填充——常见的插补策略
df["income"].fillna(df["income"].mean())     # 均值填充（mean imputation）
df["income"].fillna(df["income"].median())   # 中位数——对异常值更鲁棒
df["category"].fillna(df["category"].mode()[0])  # 最频繁的值

# 用分组均值填充——比全局均值更合理
group_means = df.groupby("dept")["salary"].transform("mean")
df["salary"] = df["salary"].fillna(group_means)

# 前向填充（forward fill）——用上一个有效值填充
# 时间序列很有用：如果周一价格已知，就用它填周二
df["price"].ffill()

# 后向填充（backward fill）
df["price"].bfill()

# 插值——在前后相邻值之间线性插值
df["temperature"].interpolate(method="linear")
    该选哪种策略？
    删除：缺失数据很少（<5%）且可能是随机的。均值/中位数填充：数值列且数据大致对称。中位数：有异常值时。众数：分类列。分组填充：缺失与另一列相关时（如"实习生"和"总监"的缺失薪资含义不同）。前向填充：时间序列数据。

    重复值检测

    Python
    # 有多少重复行？
df.duplicated().sum()

# 哪些行是重复的？（默认要求所有列都匹配）
df[df.duplicated()]

# 按特定列检查重复（例如同一 user_id = 重复）
df.duplicated(subset=["user_id"])

# keep='first'（默认）：标记除第一个以外的所有重复
# keep='last'：标记除最后一个以外的所有重复
# keep=False：标记所有出现过的重复（包括原始行）
df[df.duplicated(keep=False)]   # 看所有有孪生行的记录

# 删除重复行
df = df.drop_duplicates()
df = df.drop_duplicates(subset=["user_id"], keep="last")

    类型转换

    pandas 读 CSV 时会推断类型。这会以可预期的方式出错：数值列因为含有 "N/A" 而被读成字符串，日期被读成字符串，布尔列被读成整数。类型转换通常是加载后的第一个清洗步骤。

    Python — astype() 和 pd.to_*
    # 检查当前类型
df.dtypes

# 用 .astype() 转换
df["score"]   = df["score"].astype(float)
df["user_id"] = df["user_id"].astype(str)
df["active"]  = df["active"].astype(bool)

# pd.to_numeric——带错误处理的鲁棒转换
# errors='coerce' 把无法解析的值变成 NaN 而不是抛出异常
df["income"] = pd.to_numeric(df["income"], errors="coerce")

# pd.to_datetime——日期时间解析
df["created_at"] = pd.to_datetime(df["created_at"])
df["created_at"] = pd.to_datetime(df["date_str"], format="%Y-%m-%d")

# 一旦列变成 datetime，就解锁了 .dt 访问器：
df["year"]       = df["created_at"].dt.year
df["month"]      = df["created_at"].dt.month
df["day_of_week"] = df["created_at"].dt.day_name()   # 'Monday' 等

# 分类类型——节省内存 + 解锁排序
df["size"] = pd.Categorical(df["size"],
                             categories=["S","M","L","XL"],
                             ordered=True)

    字符串清洗

    文本列几乎总是有脏数据。Series 上的 .str 访问器让你对所有行向量化地应用任何字符串方法——不需要循环。

    Python — .str 访问器
    df["name"]  = df["name"].str.strip()          # 去除首尾空格
df["name"]  = df["name"].str.title()          # 'alice bob' → 'Alice Bob'
df["email"] = df["email"].str.lower()         # 统一小写
df["phone"] = df["phone"].str.replace("-", "", regex=False)  # 删除破折号

# 提取字符串的一部分
df["domain"] = df["email"].str.split("@").str[1]   # 从邮件里取域名
df["first"]  = df["full_name"].str.split(" ").str[0]

# 检查模式
df[df["email"].str.contains("@", na=False)]   # na=False 避免 NaN 报错
df[df["code"].str.match(r"^\d{4}$")]          # 恰好 4 位数字

# 补零填充
df["id_padded"] = df["id"].astype(str).str.zfill(6)  # "42" → "000042"

# 用正则替换
df["text"] = df["text"].str.replace(r"\s+", " ", regex=True)  # 合并多余空格

# 字符串清洗管道——链式调用
df["city"] = (df["city"]
              .str.strip()
              .str.lower()
              .str.replace(r"[^\w\s]", "", regex=True)  # 去除标点
              .str.title())

    用 IQR 检测异常值

    IQR（四分位距，Interquartile Range）方法是一种鲁棒、无分布假设的异常值标记方式。如果一个值低于 Q1 − 1.5×IQR 或高于 Q3 + 1.5×IQR，它就值得怀疑。乘数可以调整（有些领域用 3.0 标记"极端"异常值）。

    Python — IQR 异常值检测
    import pandas as pd

def flag_outliers_iqr(series: pd.Series, k: float = 1.5) -> pd.Series:
    """返回布尔掩码：True 表示该值是异常值。"""
    Q1  = series.quantile(0.25)
    Q3  = series.quantile(0.75)
    IQR = Q3 - Q1
    lower = Q1 - k * IQR
    upper = Q3 + k * IQR
    return (series < lower) | (series > upper)

# 应用到 'salary' 列
mask = flag_outliers_iqr(df["salary"])
print(f"发现异常值：{mask.sum()} 个（{mask.mean()*100:.1f}%）")
df[mask]    # 决定怎么处理之前先检查一下

# 处理异常值的几种方式：

# 1. 直接删除
df_clean = df[~mask]

# 2. 截断（Winsorisation）——用边界值替换
lower = df["salary"].quantile(0.25) - 1.5 * (df["salary"].quantile(0.75) - df["salary"].quantile(0.25))
upper = df["salary"].quantile(0.75) + 1.5 * (df["salary"].quantile(0.75) - df["salary"].quantile(0.25))
df["salary_capped"] = df["salary"].clip(lower, upper)

# 3. 对数变换——压缩大值
import numpy as np
df["salary_log"] = np.log1p(df["salary"])   # log(1+x)，对零值安全

    重命名列与重新索引

    Python
    # 重命名特定列
df.rename(columns={"old_name": "new_name", "qty": "quantity"}, inplace=True)

# 让所有列名变成小写加下划线（常见清洗步骤）
df.columns = df.columns.str.strip().str.lower().str.replace(" ", "_", regex=False)

# 重新排列列
cols = ["id", "name", "score", "grade", "dept"]
df   = df[cols]

# 过滤后重置行索引（避免 0,2,5,7... 这样的空洞）
df = df.reset_index(drop=True)   # drop=True 丢弃旧索引

# 把某列设为索引
df = df.set_index("id")   # 行现在以 'id' 为键
df = df.reset_index()     # 把索引移回一列

    建模前按顺序过一遍：(1) shape 和 dtypes——有意义吗？ (2) 缺失值——有多少，为什么？ (3) 重复项——有完全相同的行吗？ (4) 字符串列——大小写统一，没有前导空格？ (5) 日期列——解析成 datetime 了吗？ (6) 数值列——有不可能的值吗（负数年龄、200% 分数）？
