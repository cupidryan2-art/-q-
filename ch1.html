    Chapter 1

    Python Fundamentals

    Everything in Python is an object (对象). Integers, strings, functions, even None — they all live on the heap with a type, a value, and a reference count. This chapter gives you the ground floor: variables, types, strings, and operators.

    Variables and Dynamic Typing (动态类型)

    A Python variable is a name bound to an object. Assignment creates the binding; it doesn't allocate typed storage the way a C++ declaration does.

    Python
    x = 42          # x is now a name pointing at the integer object 42
x = "hello"     # x now points at a string — perfectly valid
x = [1, 2, 3]   # now a list

# Multiple assignment
a = b = c = 0   # all three names point to the same integer 0

# Tuple unpacking (元组解包) — swap without a temp variable
a, b = 10, 20
a, b = b, a     # now a=20, b=10
    C++ Comparison

    In C++, int x = 5; reserves a typed memory slot. Assigning a string to it later is a compile error.

    In Python, x = 5 creates an integer object on the heap and makes x point to it. x = "hello" creates a string object and re-points x. The integer object still exists until its reference count drops to zero and the GC reclaims it.

    Think of Python variables as std::any or a typed pointer that can be reassigned to any type.

    Naming conventions (命名规范)

    Python uses snake_case for variables and functions (not camelCase). Names starting with _ are conventionally "private." Names like __dunder__ (double underscores) are reserved for Python's own protocols.

    Core Data Types (核心数据类型)

    Python — the five scalar types
    age    = 30          # int   (整数) — arbitrary precision, no overflow
price  = 19.99       # float (浮点数) — 64-bit IEEE 754, same as C++ double
name   = "Alice"     # str   (字符串) — immutable Unicode text
active = True        # bool  (布尔值) — True or False (capital T/F!)
empty  = None        # NoneType — Python's null / void equivalent
    Noteworthy
    Python's int has arbitrary precision (任意精度) — it can hold numbers as large as your memory allows. There's no int32_t overflow: 2 ** 1000 is a valid integer. If you need fixed-width integers for interfacing with C, use the ctypes or struct module.
    C++ Comparison
    • Python int ≈ C++ bignum (not int32_t)
    • Python float ≈ C++ double
    • Python bool: True/False are actually subclasses of int — True == 1 and False == 0
    • Python None ≈ C++ nullptr or std::nullopt — but it's a real object with a type

    Type literals

    Python
    # Integer bases
dec = 255
hex_val = 0xFF     # hexadecimal
oct_val = 0o377    # octal
bin_val = 0b11111111  # binary — all equal 255

# Underscores in numbers (readability, ignored by Python)
million = 1_000_000
pi = 3.141_592_653

# String delimiters — single or double quotes, both work
s1 = 'hello'
s2 = "world"
s3 = '''multi
line
string'''          # triple quotes span newlines

    Checking Types at Runtime (类型检查)

    Python
    x = 42

# type() returns the exact type object
print(type(x))           # <class 'int'>
print(type(x) is int)    # True

# isinstance() checks for a type or its base classes — prefer this
print(isinstance(x, int))         # True
print(isinstance(x, (int, float))) # True — checks multiple types at once

# bool is a subclass of int — isinstance catches it
print(isinstance(True, int))   # True  ← might surprise you
print(type(True) is int)       # False — exact type check distinguishes them
    Best Practice
    Use isinstance() over type() == checks. It respects inheritance and is the idiomatic Python way. You'll see it constantly in library code.

    Type conversion (类型转换)

    Python — explicit casting
    int("42")         # 42    — string to int
float("3.14")     # 3.14
str(100)          # "100" — int to string
bool(0)           # False — zero is falsy
bool("hello")     # True  — non-empty string is truthy

int(3.9)          # 3     — truncates, does NOT round
round(3.9)        # 4     — rounds to nearest int

    String Operations (字符串操作)

    f-strings (格式化字符串字面量)

    Introduced in Python 3.6, f-strings are the modern way to embed expressions in strings. Prefix the string with f and put any expression inside {}.

    Python — f-strings
    name  = "Alice"
score = 98.6

# Basic embedding
print(f"Hello, {name}!")               # Hello, Alice!

# Expressions work inside {}
print(f"Score: {score:.1f}%")          # Score: 98.6%
print(f"2 + 2 = {2 + 2}")             # 2 + 2 = 4
print(f"Upper: {name.upper()}")        # Upper: ALICE

# Padding and alignment
print(f"{'left':<10}|")               # 'left      |'
print(f"{'right':>10}|")              # '     right|'
print(f"{42:05d}")                     # '00042'
print(f"{3.14159:.2f}")               # '3.14'

# Debug shorthand (Python 3.8+)
x = 42
print(f"{x=}")                         # x=42
    C++ Comparison
    f-strings are essentially Python's version of C++20 std::format("{}", value) or the older sprintf, but inline and evaluated at runtime. Unlike printf, there's no separate format string — the expression is inside the string literal.

    Common string methods

    Python — string methods (strings are immutable — all methods return new strings)
    s = "  Hello, World!  "

s.strip()           # "Hello, World!"   — remove leading/trailing whitespace
s.lower()           # "  hello, world!  "
s.upper()           # "  HELLO, WORLD!  "
s.replace("World", "Python")  # "  Hello, Python!  "
s.split(",")        # ['  Hello', ' World!  ']
", ".join(["a", "b", "c"])    # "a, b, c"

s.startswith("  H")  # True
s.endswith("  ")     # True
s.strip().find("World")  # 7  — index of first match, -1 if not found
"hello" in s.lower() # True  — membership test

    String slicing (切片)

    Python's slice syntax s[start:stop:step] is one of its most expressive features. All indices are optional.

    Python — slicing
    s = "Hello, World!"
#    0123456789...

s[0]       # 'H'          — index from start
s[-1]      # '!'          — index from end
s[7:12]    # 'World'      — [start, stop)
s[:5]      # 'Hello'      — from beginning to index 5
s[7:]      # 'World!'     — from index 7 to end
s[::2]     # 'Hlo ol!'    — every other character (step=2)
s[::-1]    # '!dlroW ,olleH'  — reverse the string
    C++ Comparison
    Python slicing is similar to std::string::substr(start, length), but more powerful: negative indices, step parameter, and the stop index is exclusive (like STL iterators). Note that Python strings are immutable — s[0] = 'h' raises a TypeError. Use s = 'h' + s[1:] or convert to a list first.

    Operators (运算符)

    Arithmetic operators (算术运算符)

    Python
    10 + 3    # 13   — addition
10 - 3    # 7    — subtraction
10 * 3    # 30   — multiplication
10 / 3    # 3.333...  — TRUE division (always returns float)
10 // 3   # 3    — FLOOR division (integer result, truncates toward -inf)
10 % 3    # 1    — modulo (remainder)
2 ** 10   # 1024 — exponentiation (no std::pow needed)
    Watch Out
    In Python 3, / is always true division — 7 / 2 is 3.5, not 3. Use // for integer (floor) division. This is different from C++, where 7 / 2 == 3 for integer operands.

    Comparison and logical operators

    Python
    # Comparison (比较运算符)
5 == 5     # True
5 != 4     # True
5 > 3      # True
5 <= 5     # True

# Python allows chained comparisons — very readable
1 < x < 10       # equivalent to x > 1 and x < 10
0 <= age < 120   # range check in one expression

# Logical (逻辑运算符) — English keywords, not symbols
True and False   # False
True or False    # True
not True         # False

# Identity vs equality
a = [1, 2, 3]
b = [1, 2, 3]
a == b     # True  — same VALUE
a is b     # False — different OBJECTS in memory
a is None  # False
x = None
x is None  # True  — use 'is' for None checks, not ==
    C++ Comparison
    Python's and/or/not are equivalent to C++'s &&/||/!. Python's is operator checks pointer equality (like comparing addresses), similar to ptr1 == ptr2 for raw pointers, not the values they point to.

    Input and Output (输入输出)

    Python — print()
    print("Hello")                    # Hello\n (newline by default)
print("a", "b", "c")             # a b c   (space separator by default)
print("a", "b", "c", sep="-")    # a-b-c
print("no newline", end="")       # suppress the trailing \n
print()                           # just prints a newline

# Printing multiple types — no conversion needed
x = 42
print("Value:", x, "type:", type(x))  # Value: 42 type: <class 'int'>
    Python — input()
    name = input("Enter your name: ")   # always returns a str
age  = int(input("Enter your age: "))   # cast immediately if you need int

print(f"Hello {name}, you are {age} years old.")
    Key Takeaway
    input() always returns a string (字符串). If you expect a number, wrap it: int(input(...)) or float(input(...)). Forgetting this is one of the most common beginner errors.
