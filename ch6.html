    第 6 章

    文件 I/O 与数据格式

    每个真实的数据科学项目都从某处加载数据开始——文本文件、CSV 导出、JSON API 响应、Excel 表格。 Python 把这件事处理得异常顺手。这一章带你从最底层（原始文件句柄）走到你每天真正会用的写法。

    读写文本文件

    open() 函数

    open(path, mode) 返回一个文件对象（file object）。mode 参数控制你是读、写还是追加，以及文件是文本模式还是二进制模式。

    模式含义文件必须存在？
    'r'读文本（默认）必须——否则 FileNotFoundError
    'w'写文本——覆盖已有内容不必——不存在则创建
    'a'追加文本——在末尾添加不必——不存在则创建
    'x'独占创建——文件已存在则失败不必——必须是新文件
    'rb' / 'wb'读/写二进制字节同上
    Python — 读文件（低层写法）
    f = open("notes.txt", "r", encoding="utf-8")
content = f.read()    # 整个文件作为一个字符串
f.close()             # 必须关闭——释放文件句柄

# 逐行读取
f = open("notes.txt", "r", encoding="utf-8")
for line in f:           # 文件对象是可迭代的
    print(line.strip())  # strip() 去掉末尾的 \n
f.close()
    始终指定编码
    明确传入 encoding="utf-8"。默认编码因操作系统而异（macOS/Linux 是 UTF-8，Windows 常常是 CP1252）。显式指定让你的代码可移植，也避免非 ASCII 文本的乱码。

    with 语句 — 上下文管理器（context manager）

    手动调用 f.close() 容易出错——如果 open() 和 close() 之间抛了异常，文件就一直开着。Python 的 with 语句保证代码块退出时文件一定会被关闭，不管是正常退出还是因为异常。

    永远用 with open(...)。你不会在有经验的 Python 开发者的生产代码里看到裸的 open()。

    Python — with open()（正确的惯用写法）
    # 读取
with open("notes.txt", "r", encoding="utf-8") as f:
    content = f.read()          # 整个文件作为字符串
# 这里 f 自动关闭，即使发生了异常也一样

# 把所有行读入一个 list
with open("notes.txt", "r", encoding="utf-8") as f:
    lines = f.readlines()       # ['行1\n', '行2\n', ...]

# 逐行遍历（对大文件内存友好）
with open("notes.txt", "r", encoding="utf-8") as f:
    for line in f:
        process(line.strip())

# 写入——覆盖已有内容
with open("output.txt", "w", encoding="utf-8") as f:
    f.write("Hello, World!\n")
    f.write("第二行\n")

# 一次性写入多行
lines = ["苹果\n", "香蕉\n", "樱桃\n"]
with open("fruits.txt", "w", encoding="utf-8") as f:
    f.writelines(lines)

# 追加——不清空原有内容
with open("log.txt", "a", encoding="utf-8") as f:
    f.write("新的日志条目\n")
    C++ 对比
    Python 的 with 块等价于 C++ RAII。文件对象实现了 __enter__ 和 __exit__——上下文管理器协议——所以 __exit__ 会自动调用 close()。这就像栈上的 std::ifstream，析构函数在它离开作用域时关闭文件——只不过它适用于任何资源，不只是文件（数据库连接、锁、网络 socket 等等）。

    CSV 文件

    CSV（逗号分隔值，Comma-Separated Values）是最常见的纯文本数据格式。下面看看从手动解析到正确工具的演进过程。

    阶段一 — 手动解析（生产代码别这么做）

    Python — 朴素 CSV 解析
    # 这种写法在以下情况会出错：带引号的字段、引号里的逗号、字段里的换行...
with open("data.csv", "r", encoding="utf-8") as f:
    header = f.readline().strip().split(",")
    rows = []
    for line in f:
        values = line.strip().split(",")
        rows.append(dict(zip(header, values)))

    阶段二 — csv 模块（处理边缘情况）

    Python — csv 模块
    import csv

# 用 DictReader 读取——每行是一个以表头为键的 dict
with open("students.csv", "r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f)
    students = list(reader)
    # [{'name': 'Alice', 'score': '92'}, {'name': 'Bob', 'score': '78'}, ...]

# 注意：所有值都是字符串——需要自己转换类型
scores = [int(s["score"]) for s in students]

# 用 DictWriter 写入
fieldnames = ["name", "score", "grade"]
with open("results.csv", "w", encoding="utf-8", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=fieldnames)
    writer.writeheader()
    writer.writerows([
        {"name": "Alice", "score": 92, "grade": "A"},
        {"name": "Bob",   "score": 78, "grade": "C"},
    ])
    为什么要 newline=""？
    用 csv 模块打开 CSV 文件时，传 newline=""。否则模块自己的换行处理会和 Python 的通用换行转换冲突，在 Windows 上产生空行。

    阶段三 — pandas（你实际会用的方式）

    Python — pandas read_csv（一行代替上面所有）
    import pandas as pd

# 一行：加载、解析表头、推断类型、处理编码
df = pd.read_csv("students.csv")

print(df.head())        # 前 5 行
print(df.dtypes)        # 列类型（自动推断！）
print(df["score"].mean())   # 列操作

# 过滤和排序
top_students = df[df["score"] >= 90].sort_values("score", ascending=False)

# 保存回 CSV
top_students.to_csv("top_students.csv", index=False)
    该用哪个
    写脚本且不需要依赖、数据简单时用 csv 模块。做任何分析、过滤或变换时用 pandas——它的使用体验好到没得比。对于 prototype 之外的场景，避免手动 split(",")。

    JSON 文件

    JSON 是 API 响应和配置文件的主流格式。Python 的 json 模块把 JSON 直接映射到 Python 对象：对象变 dict、数组变 list、字符串是 str、数字是 int 或 float、布尔是 bool、null 变 None。

    Python — 读取 JSON
    import json

# 从文件——json.load()（注意：没有 's'）
with open("config.json", "r", encoding="utf-8") as f:
    config = json.load(f)    # 返回 dict、list 等

print(config["database"]["host"])

# 从字符串——json.loads()（'s' = string）
raw = '{"name": "Alice", "scores": [90, 85, 92]}'
data = json.loads(raw)
print(data["scores"])        # [90, 85, 92]
    Python — 写入 JSON
    import json

results = {
    "experiment": "run_01",
    "params": {"lr": 0.001, "epochs": 100},
    "accuracy": 0.9423,
    "losses": [1.2, 0.8, 0.5, 0.3],
}

# 写到文件——json.dump()
with open("results.json", "w", encoding="utf-8") as f:
    json.dump(results, f, indent=2)   # indent=2 做格式化输出

# 写到字符串——json.dumps()（'s' = string）
s = json.dumps(results, indent=2)
print(s)

# 处理不可序列化的类型
import datetime
data = {"timestamp": datetime.datetime.now()}
# json.dumps(data)  ← 抛 TypeError: datetime 不可序列化

# 解决方案：用 default= 参数指定序列化函数
def json_default(obj):
    if isinstance(obj, datetime.datetime):
        return obj.isoformat()
    raise TypeError(f"不可序列化：{type(obj)}")

json.dumps(data, default=json_default)
    C++ 对比
    Python 的 json 模块做的是 C++ 里 nlohmann/json 或 RapidJSON 做的事，但没有任何样板代码。不需要定义 schema，不需要定义映射类型——Python 的动态类型直接对应 JSON 的类型。json.loads(s) 的 C++ 等价是 auto j = nlohmann::json::parse(s);，但访问嵌套字段在 Python 里轻松得多：data["key"]["nested"] 直接就能用。

    Excel 文件（.xlsx）

    Excel 文件是二进制格式——你没法用 open() 读它。需要 openpyxl 做底层单元格操作，或者用 pandas 做数据分析。

    Python — pandas 处理 Excel（最常用）

# 读取第一个 sheet
df = pd.read_excel("report.xlsx")

# 按名称或索引读取特定 sheet
df = pd.read_excel("report.xlsx", sheet_name="Sales Q1")

df.to_excel("output.xlsx", sheet_name="Results", index=False)

    综合实战 — 一个迷你 ETL 管道

    ETL（Extract, Transform, Load——提取、变换、加载）是数据工程的骨架：从某处读取数据，清洗或重塑，写到某处。下面是一个综合本章所学的小例子。

    Python — 读 CSV → 变换 → 写 JSON
    import csv
import json
from pathlib import Path

def load_students(path: str) -> list[dict]:
    """读取 CSV，返回带类型化值的字典列表。"""
    students = []
    with open(path, "r", encoding="utf-8", newline="") as f:
        for row in csv.DictReader(f):
            students.append({
                "name":  row["name"].strip(),
                "score": int(row["score"]),
                "grade": row["grade"].strip(),
            })
    return students

def add_pass_fail(students: list[dict]) -> list[dict]:
    """纯函数——返回新列表，添加及格/不及格字段。"""
    return [
        {**s, "passed": s["score"] >= 60}
        for s in students
    ]

def save_json(data, path: str) -> None:
    Path(path).parent.mkdir(parents=True, exist_ok=True)
    with open(path, "w", encoding="utf-8") as f:
        json.dump(data, f, indent=2, ensure_ascii=False)

# 运行管道
students = load_students("students.csv")
students = add_pass_fail(students)
save_json(students, "output/students.json")

print(f"处理了 {len(students)} 名学生")
