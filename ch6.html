    File I/O & Data Formats 文件操作

    Every real data science project starts with loading data from somewhere — a text file, a CSV export, a JSON API response, an Excel spreadsheet. Python makes this remarkably straightforward. This chapter takes you from the lowest level (raw file handles) to the idiom you'll actually use every day.

    Reading and Writing Text Files

    The open() function

    open(path, mode) returns a file object (文件对象). The mode argument controls whether you read, write, or append, and whether the file is treated as text or binary.

    ModeMeaningFile must exist?
    'r'Read text (default)Yes — FileNotFoundError otherwise
    'w'Write text — overwrites existing contentNo — creates if missing
    'a'Append text — adds to endNo — creates if missing
    'x'Exclusive create — fails if file existsNo — must be new
    'rb' / 'wb'Read / write binary bytesSame as above
    Python — reading a file (low-level)
    f = open("notes.txt", "r", encoding="utf-8")
content = f.read()    # entire file as one string
f.close()             # MUST close — releases the file handle

# Read line by line
f = open("notes.txt", "r", encoding="utf-8")
for line in f:           # file objects are iterable
    print(line.strip())  # strip() removes trailing \n
f.close()
    Always Specify Encoding
    Pass encoding="utf-8" explicitly. The default encoding varies by OS (UTF-8 on macOS/Linux, often CP1252 on Windows). Specifying it makes your code portable and avoids silent mojibake (乱码) errors with non-ASCII text.

    The with Statement — Context Managers (上下文管理器)

    Calling f.close() manually is error-prone — if an exception occurs between open() and close(), the file stays open. Python's with statement guarantees the file is closed when the block exits, regardless of whether it exits normally or via an exception.

    Always use with open(...). You will never see experienced Python developers using bare open() in production code.

    Python — with open() (the correct idiom)
    # Reading
with open("notes.txt", "r", encoding="utf-8") as f:
    content = f.read()          # entire file as string
# f is automatically closed here, even if an exception was raised

# Read all lines into a list
with open("notes.txt", "r", encoding="utf-8") as f:
    lines = f.readlines()       # ['line1\n', 'line2\n', ...]

# Iterate line by line (memory-efficient for large files)
with open("notes.txt", "r", encoding="utf-8") as f:
    for line in f:
        process(line.strip())

# Writing — overwrites existing content
with open("output.txt", "w", encoding="utf-8") as f:
    f.write("Hello, World!\n")
    f.write("Second line\n")

# Writing multiple lines at once
lines = ["apple\n", "banana\n", "cherry\n"]
with open("fruits.txt", "w", encoding="utf-8") as f:
    f.writelines(lines)

# Appending — adds to end without erasing
with open("log.txt", "a", encoding="utf-8") as f:
    f.write("New log entry\n")
    C++ Comparison
    Python's with block is the equivalent of C++ RAII. The file object implements __enter__ and __exit__ — the context manager protocol — so __exit__ calls close() automatically. This is like a std::ifstream on the stack whose destructor closes the file when it goes out of scope, except it works for any resource, not just files (database connections, locks, network sockets, etc.).

    Reading structured text

    Python — common text file patterns
    # Count lines
with open("data.txt", "r", encoding="utf-8") as f:
    line_count = sum(1 for _ in f)

# Find lines matching a pattern
with open("server.log", "r", encoding="utf-8") as f:
    errors = [line for line in f if "ERROR" in line]

# Read a config file (key=value format)
config = {}
with open("config.ini", "r", encoding="utf-8") as f:
    for line in f:
        line = line.strip()
        if "=" in line and not line.startswith("#"):
            key, _, value = line.partition("=")
            config[key.strip()] = value.strip()

    CSV Files (逗号分隔值)

    CSV (Comma-Separated Values) is the most common plain-text data format. Let's see the evolution from doing it manually to using the right tool for the job.

    Stage 1 — Manual parsing (don't do this in production)

    Python — naive CSV parsing
    # This breaks on: quoted fields, commas inside quotes, newlines in fields...
with open("data.csv", "r", encoding="utf-8") as f:
    header = f.readline().strip().split(",")
    rows = []
    for line in f:
        values = line.strip().split(",")
        rows.append(dict(zip(header, values)))

    Stage 2 — The csv module (handles edge cases)

    Python — csv module
    import csv

# Reading with DictReader — each row is a dict keyed by header
with open("students.csv", "r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f)
    students = list(reader)
    # [{'name': 'Alice', 'score': '92'}, {'name': 'Bob', 'score': '78'}, ...]

# Note: all values are strings — cast as needed
scores = [int(s["score"]) for s in students]

# Writing with DictWriter
fieldnames = ["name", "score", "grade"]
with open("results.csv", "w", encoding="utf-8", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=fieldnames)
    writer.writeheader()
    writer.writerows([
        {"name": "Alice", "score": 92, "grade": "A"},
        {"name": "Bob",   "score": 78, "grade": "C"},
    ])
    Why newline=""?
    Pass newline="" when opening CSV files with the csv module. Otherwise the module's own newline handling conflicts with Python's universal newline translation, producing blank rows on Windows.

    Stage 3 — pandas (what you'll actually use)

    Python — pandas read_csv (one line replaces all of the above)
    import pandas as pd

# One line to load, parse headers, infer types, handle encoding
df = pd.read_csv("students.csv")

print(df.head())        # first 5 rows
print(df.dtypes)        # column types (automatically inferred!)
print(df["score"].mean())   # column operations

# Filtering and sorting
top_students = df[df["score"] >= 90].sort_values("score", ascending=False)

# Save back to CSV
top_students.to_csv("top_students.csv", index=False)
    When to Use Which
    Use the csv module when you're writing a script that needs no dependencies and the data is simple. Use pandas when you're doing any analysis, filtering, or transformation — the ergonomics are incomparably better. Avoid manual split(",") for anything beyond a quick prototype.

    JSON Files (JavaScript Object Notation)

    JSON is the dominant format for API responses and configuration files. Python's json module maps JSON directly to Python objects: objects become dict, arrays become list, strings stay str, numbers become int or float, booleans become bool, and null becomes None.

    Python — reading JSON
    import json

# From a file — json.load() (note: no 's')
with open("config.json", "r", encoding="utf-8") as f:
    config = json.load(f)    # returns a dict, list, etc.

print(config["database"]["host"])

# From a string — json.loads() ('s' = string)
raw = '{"name": "Alice", "scores": [90, 85, 92]}'
data = json.loads(raw)
print(data["scores"])        # [90, 85, 92]
    Python — writing JSON
    import json

results = {
    "experiment": "run_01",
    "params": {"lr": 0.001, "epochs": 100},
    "accuracy": 0.9423,
    "losses": [1.2, 0.8, 0.5, 0.3],
}

# To a file — json.dump()
with open("results.json", "w", encoding="utf-8") as f:
    json.dump(results, f, indent=2)   # indent=2 for pretty printing

# To a string — json.dumps() ('s' = string)
s = json.dumps(results, indent=2)
print(s)

# Handle non-serialisable types
import datetime
data = {"timestamp": datetime.datetime.now()}
# json.dumps(data)  ← raises TypeError: datetime is not JSON serialisable

# Fix: use default= to specify a serialiser
def json_default(obj):
    if isinstance(obj, datetime.datetime):
        return obj.isoformat()
    raise TypeError(f"Not serialisable: {type(obj)}")

json.dumps(data, default=json_default)
    C++ Comparison
    Python's json module does what nlohmann/json or RapidJSON do in C++, but with zero boilerplate. There's no schema to define, no type to map to — Python's dynamic types map directly to JSON's types. The equivalent of json.loads(s) in C++ would be auto j = nlohmann::json::parse(s);, but accessing nested fields is far more ergonomic in Python: data["key"]["nested"] just works.

    JSON with pandas

    Python — JSON + pandas
    import pandas as pd

# Load a JSON file of records directly into a DataFrame
df = pd.read_json("students.json")

# Load from a URL (pandas handles the HTTP request)
df = pd.read_json("https://api.example.com/data.json")

# Convert DataFrame back to JSON
df.to_json("output.json", orient="records", indent=2)
# orient="records" → list of dicts [{col: val, ...}, ...]

    Excel Files (.xlsx)

    Excel files are binary formats — you can't read them with open(). You'll need either openpyxl for low-level cell manipulation or pandas for data analysis.

    Terminal — install dependencies
    pip install openpyxl   # required by pandas for .xlsx support
    Python — Excel with pandas (most common)
    import pandas as pd

# Read the first sheet
df = pd.read_excel("report.xlsx")

# Read a specific sheet by name or index
df = pd.read_excel("report.xlsx", sheet_name="Sales Q1")
df = pd.read_excel("report.xlsx", sheet_name=0)

# Read multiple sheets — returns a dict of DataFrames
sheets = pd.read_excel("report.xlsx", sheet_name=None)
for name, df in sheets.items():
    print(f"Sheet '{name}': {len(df)} rows")

# Write to Excel
df.to_excel("output.xlsx", sheet_name="Results", index=False)

# Write multiple sheets
with pd.ExcelWriter("multi_sheet.xlsx", engine="openpyxl") as writer:
    df_sales.to_excel(writer, sheet_name="Sales",    index=False)
    df_costs.to_excel(writer, sheet_name="Costs",    index=False)
    df_summary.to_excel(writer, sheet_name="Summary", index=False)

    Putting It Together — A Mini ETL Pipeline

    ETL (Extract, Transform, Load) is the backbone of data engineering: read data from a source, clean or reshape it, write it somewhere else. Here's a small example using what we've covered.

    Python — read CSV → transform → write JSON
    import csv
import json
from pathlib import Path

def load_students(path: str) -> list[dict]:
    """Read a CSV and return a list of dicts with typed values."""
    students = []
    with open(path, "r", encoding="utf-8", newline="") as f:
        for row in csv.DictReader(f):
            students.append({
                "name":  row["name"].strip(),
                "score": int(row["score"]),
                "grade": row["grade"].strip(),
            })
    return students

def add_pass_fail(students: list[dict]) -> list[dict]:
    """Pure function — returns new list with pass/fail field added."""
    return [
        {**s, "passed": s["score"] >= 60}
        for s in students
    ]

def save_json(data, path: str) -> None:
    Path(path).parent.mkdir(parents=True, exist_ok=True)
    with open(path, "w", encoding="utf-8") as f:
        json.dump(data, f, indent=2, ensure_ascii=False)

# Run the pipeline
students = load_students("students.csv")
students = add_pass_fail(students)
save_json(students, "output/students.json")

print(f"Processed {len(students)} students")
